phase_0

  • Purpose: Push x29 (frame pointer) and x30 (link register, return address) onto the stack. This step saves the caller’s frame pointer and return address for later restoration.
  • Purpose: Assign the current stack pointer sp to the frame pointer x29, i.e., establish a new stack frame for the current function.

From now on, x29 points to the bottom of the stack for this function call, making it convenient to access local variables or pass parameters later.

Then, the function first calls the read_int function to read an int value, stores the return value in w0, and then compares it with the value at 0x4a0000+0x54(84)=0x4a0054 to see if they are equal. You can use the gdb examine command to view the value at this memory location:

It can be seen that it is 2022.

Then there is a cmp instruction comparing the input value (stored in x0) with 2022 (stored in x1). If they are not equal, it jumps to the line calling the explode function, so they must be equal. That is, the input value must be 2022.

phase_1

phase_1 and phase_0 are similar.

It can be seen that the function loads the value at address 0x4a0000+0x58(88)=0x4a0058 into x1, then calls strcmp, whose parameters should be x0 and x1. x0 is our input value, meaning it compares the string we input with the string at address 0x4a0058, and checks if the return value w0 is 0. If not 0, it jumps to the line calling explode.

It should be noted that the address 0x4a0058 does not store a string, but the address of a string. So you need to first read this address, then read the string at that string address.

The answer is: “Fault Tolerance: Reliable Systems from Unreliable Components.”

phase_2

The first line allocates 64 bytes of stack space, which can hold 64/8=8 64-bit values. The first line places x29 at sp and x30 at sp+8.

Then save sp to the x29 register.

Then place the values of registers x19 and x20 at sp+16 and sp+24, i.e.,

  • sp —> x29
  • sp+8 —> x30
  • sp+16 —> x19
  • sp+24 —> x20

Then x1 = sp+0x20 = sp+32, as the second argument of read_8_numbers, then call read_8_the function numbers.

It can be seen that this function first allocates 0x20, i.e., 32 bytes of stack space sp=sp-32, then stores x29 at sp+16 and x30 at sp+24.

Then set the stack frame pointer x29 to sp+0x10, i.e., sp+16.

Then put x1 into x2, x1 is the second argument of read_8_numbers.

Then, x1 = x1+0x1c = x1+28.

Store x1 at sp+8.

x1 = x2 + 0x18 =x1 + 24

Store x1 at sp.

x1 is the value at address 0x464000+0x858, a pointer. It is clearly the starting address of an array, and it can be seen as a format string, used as __the second parameter of isoc99_scanf

the first parameter is x0, which has remained unchanged since returning from read_line

after the function call ends, compare the return value with 0x7; if it is less than or equal, it explodes, so you must input 8 numbers.
