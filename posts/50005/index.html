WSL2 Archlinux镜像网络模式无法访问网络的问题

even629
2025-10-03 (Updated: 2025-10-03)
Misc
Misc, WSL
时间轴

时间轴

2025-10-03

init

本文介绍了 Win11 下 WSL2 Archlinux 镜像网络模式重启后可能无法访问网络的问题，并总结了通过排查网关连通性来区分 DNS 和防火墙故障，进而提供关闭 DNS 自动生成、放行 wsl.exe 防火墙权限及切换网络模式等具体解决方法。

在 Win11 + WSL2 archlinux 的 mirrored 网络模式下，重启后有概率会触发无法 WSL 访问网络但 Windows 可以访问的问题。

wsl 配置

ping 网关

先确认能不能 ping 通网关。

  • 如果能 ping 通 IP 但不能访问域名 → 是 DNS 问题
  • 连 IP 都 ping 不通 → 是 网络转发 / 网卡 / 防火墙问题

DNS 问题

为避免每次重启被覆盖，可以关闭自动生成

重启 WSL

查看网络是否恢复

防火墙问题

打开 windows 安全中心的防火墙和网络保护：


点击允许应用通过防火墙，将 C:\Windows\System32\wsl.exe 加入。再次查看网络是否恢复。

其他

如果出现以下报错：

  1. 可以尝试重启
  1. 尝试更新WSL后重试

  1. 或者在WSL Settings中先切换到NAT模式，然后能成功打开wsl后切换回Mirror模式

  2. 如果关闭了DNS自动生成，可尝试打开即设置/etc/wsl.conf中的generateResolvConf = true
