Issue with WSL2 Archlinux Mirrored Network Mode Unable to Access Network

This article introduces the issue where WSL2 Archlinux under Win11 may be unable to access the network after restarting in mirrored network mode, and summarizes methods to distinguish DNS and firewall faults by checking gateway connectivity, then provides specific solutions such as disabling automatic DNS generation, allowing wsl.exe firewall permissions, and switching network modes.

Under the mirrored network mode of Win11 + WSL2 Archlinux, there is a probability after restart that WSL cannot access the network while Windows can.

WSL Configuration

Ping Gateway

First confirm whether the gateway can be pinged.

  • If IP can be pinged but domain names cannot be accessed → it is a DNS issue
  • If even the IP cannot be pinged → it is a network forwarding / network card / firewall issue

DNS Issue

To avoid being overwritten on each restart, you can disable auto-generation

Restart WSL

Check if the network is restored

Firewall issue

Open Windows Security Center’s Firewall & network protection:


Click ‘Allow an app through firewall’ and add C:\Windows\System32\wsl.exe.
Check again if the network is restored.

Other

If the following error occurs:

  1. You can try restarting
  1. Try updating WSL and then retry

  1. Or in WSL Settings, first switch to NAT mode, then after successfully opening WSL, switch back to Mirror mode

  2. If DNS auto-generation is disabled, try enabling it, i.e., set it/etc/wsl.confin; among; duringgenerateResolvConf = true
