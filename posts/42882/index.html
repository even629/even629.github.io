scanf

Difference between %c and whitespace

Will read a character exactly as it is, including spaces, tabs, and newlines \n.

Because there is a space in the format string, scanf will first skip all whitespace characters (space, \t, \n) before reading a valid character.

Summary:

  • scanf returns EOF when a matching failure occurs or the end of the file is reached
  • %d, %f and other numeric format specifiers: automatically skip whitespace by default.
  • %c: does not skip whitespace; " %c" is needed to ignore spaces and newlines.
  • Principle: In scanf’s format string, whitespace characters (space, \n, \t) mean “match any amount of whitespace”.

getline

The return value of getline() is the number of characters read (including the newline character, excluding the null terminator\0); it returns -1 on failure. getline is a POSIX function, not a standard C function.

ParameterMeaning
char **lineptrPoints to achar*Pointer (used to store the read string, the function automatically allocates/expands memory internally)
size_t *nPoints to the buffer size (initially 0), the function dynamically allocates/expands memory as needed
FILE *streamInput stream, such asstdin, file handles, etc.

Return value

  • Success: returns the number of characters read (including the newline character\n, but not including the null terminator\0）
  • Failure: returns-1, and seterrno

Using getline requires including <stdio.h> and <stdlib.h>, but more importantly, getline() is unavailable on some non-POSIX platforms (such as Windows or older standard C compilers).

.precisionDescription

strdup

Equivalent to strcpy but automatically allocates memory

strtok

Parameter
str: The string to be split. On the first call, pass the string to be split; on subsequent calls, pass NULL to indicate continuing to split the same string.
delim: The delimiter string. strtok() will split str based on any character in this string.

Return value
Returns a pointer to the next token. If there are no more tokens, it returns NULL.

Notes

  1. Modifies the original string: strtok() modifies the passed string by replacing the delimiter with \0 (null character). Therefore, the original string is destroyed.
  2. Non-reentrant: strtok() uses a static buffer to save state, so it is not thread-safe. If used in a multithreaded environment, consider using strtok_r() (reentrant version).
  3. Consecutive delimiters: If there are consecutive delimiters in the string, strtok() will ignore them and return the next valid token.

Reentrant version: strtok_r()
strtok_r() is the reentrant version of strtok(), which allows you to use it safely in a multithreaded environment. Its prototype is as follows:

char *strtok_r(char *str, const char *delim, char *saveptr);
saveptr: A pointer to char used to save the splitting state.

strcspn

strcspn is a string handling function in the C standard library <string.h>, used to find the position of the first character in the target string that matches any character in the specified set.

  • Function prototype
  • Function description

It returns the position (index) of the first character in string s that contains any character in reject. If s does not contain any character in reject, it returns strlen(s).

💡 “cspn” stands for complement span, meaning: returns the length of the “longest prefix not in reject”

strspn

strspn is a function in the C standard library <string.h>, used to calculate how many characters at the beginning of a string all belong to the specified character set.
The function name strspn comes from the abbreviation of “string span”, meaning “the span of the string” or “the length of the continuously satisfying prefix in the string”.

Compare with strcspn

Function nameMeaning description
strspnspan of characters in accept
strcspnspan of characters not in reject

In other words:

  • strspn(s, accept): Starting from the beginning, counts how many characters are in accept

  • strcspn(s, reject): Starting from the beginning, counts how many characters are not in reject

  • Function prototype

  • Function description

strspn(s, accept) returns the number of consecutive characters at the beginning of string s that all appear in accept.

  1. It does not skip characters, nor does it check the entire string;
  2. Once it encounters a character that does not belong to accept, it stops counting;
  3. The return value is of type size_t (i.e., an unsigned integer), representing the length of the match.
  • Example

  • ✨ Analysis:
    String s = “abcabc123”
    Starts with “a”, “b”, “c”, exactly 6 consecutive characters are all in “abc”
    The 7th character is ‘1’, which is not in “abc” → counting stops
    So it returns 6

  • Application scenarios
    📌 Check if the beginning of a string contains only certain characters

📌 Skip all valid characters in the prefix
