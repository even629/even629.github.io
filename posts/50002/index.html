This article introduces the basic and extended modes of GCC inline assembly, discusses in detail its syntax format, input/output constraints, clobber list, goto jumps, and other core mechanisms, and summarizes advanced usage and common pitfalls of implementing low-level optimization operations such as memcpy, memset, and system register read/write by combining C language macros and statement expressions.

Inline Assembly Language: Embedding assembly code in C language

Purpose:

  • Optimization: Optimize for specific time-sensitive code
  • C language needs to access certain special instructions to implement special functions, such as memory barrier instructions

Two Modes of Inline Assembly

  • Basic Inline Assembly
  • Extended Inline Assembly

Basic Inline Assembly

Format:

  • asm keyword: Indicates this is a GNU extension

  • Qualifiers

    • volatile: This qualifier is usually not needed in basic inline assembly
    • inline: Inline, the asm assembly code will be as small as possible

  • Assembler Instructions

    • The GCC compiler treats inline assembly as a string

    • GCC does not parse or analyze inline assembly

    • For multiple assembly instructions, use “\n\t” to create new lines

    • GCC’s optimizer can rearrange the order of assembly instructions. If you need to preserve the order, use multiple inline assembly blocks

Extended inline assembly

  • Format
    • asm keyword: Indicates this is a GNU extension
    • asm-qualifiers
      • volatile: Used to disable GCC optimization
      • inline: the asm assembly code will be as small as possible
      • goto jumps to a C language label after the inline assembly ends

  • output section:

used to describe in the instruction sectionC language variables that can be modifiedand constraint conditions

  • each output constraint usually starts with “=”, followed by a letter indicating the operand type, then constraints on variable binding
  • the output section typically uses “=” or “+” as output constraints, where**“=” indicates the modified operand is write-only**，”“+” indicates the modified operand is readable and writable
  • the output section can be empty

commonly used constraint modifiers

  • ‘r’ A register operand is allowed provided that it is in a general register.
  • input section

used to describe in the instruction sectionC language variable that can only be readand constraints

  • The parameters described in the input section are read-only. Do not attempt to modify the content of the input section parameters, because the GCC compiler assumes that the content of the input section parameters remains consistent before and after the inline assembly.

  • In the input section, the “=” or “+” constraints cannot be used, otherwise the compiler will report an error.

  • The input section can be empty

  • Clobbers

  • The assembly code maymodify registers other than the outputs。
  • If not told to the compiler, the compiler will assume these registers remain unchanged, potentially leading to errors or unpredictable behavior.
  • Clobber is also a type ofmemory barrier(especially"memory"), ensuring the compiler does not reorder or cache variables.
NameFunction
"x0","x1", …General-purpose registers are modified
"cc"Condition code register (flags) is modified
"memory"Assembly accesses memory, ensuring the compiler flushes registers to memory and reloads them
"redzone"Uses stack space in the x86-64 redzone area
  • “memory” tells the GCC compiler that the inline assembly instruction changes values in memory, forcing the compiler to store all cached values before executing the assembly code and reload them after execution, to prevent compiler reordering
  • “cc” indicates that the inline assembly code modifies status register-related flags

  • It is not allowed to write in the clobber listStack pointer register (esp/rsp)。

  • Do not list output registers repeatedly.

  • The four possible cases can be combined, separated by commas

  • Parameter representation in the instruction part
    • %0 corresponds to the first parameter of the output/input part, %1 represents the second parameter

Inline assembly: goto

The goto template of inline assembly can jump to a C language label.

  • The output section of the goto template must be empty.
  • Add a gotolabels section that lists C language labels, which are labels allowed for jumping.
