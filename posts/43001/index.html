This article introduces the basic concepts of data structures and algorithms, including the logical structure of data (linear structures such as linear lists, stacks, queues, and nonlinear structures such as trees and graphs) and storage structures (sequential, linked, indexed, and hash storage), as well as the definition of algorithms (finiteness, determinism, feasibility, input/output) and efficiency measurement (time complexity). It also discusses in detail the sequential representation and linked representation of linear lists (singly linked lists, doubly linked lists, circular linked lists, static linked lists), as well as the sequential storage and linked storage structures of stacks and their characteristics.

Storage Structure of Data

Storage structure refers to the representation of data structures in a computer, also known as physical structure.

  • Sequential Storage

Sequential storage means storing logically adjacent elements in physically adjacent storage units;

  • Linked Storage

Linked storage uses pointers indicating the storage address of elements to represent the logical relationships between elements;

  • Index storage

Index storage refers to building an additional index table while storing element information. Each item in the index table is called an index entry, and the general form of an index entry is (key, address);

  • Hash storage

Hash storage directly calculates the storage address of an element based on its key, also known as Hash storage.

Basic concepts of algorithms

  • Finiteness

Finiteness means that an algorithm must always terminate after executing a finite number of steps, and each step must be completed within a finite time;

  • Determinism

Determinism means that every instruction in an algorithm must have a precise meaning, and the same input can only yield the same output;

  • Feasibility

Feasibility means that the operations described in an algorithm can be implemented by executing already implemented basic operations a finite number of times;

  • Input

Input refers to an algorithm having 0 or more inputs, which are taken from a set of specific objects

  • Output

Output refers to an algorithm having one or more outputs, which are quantities that have a specific relationship with the inputs.

Head node

Usually, ahead pointeris used to identify a singly linked list; a NULL head pointer indicates an empty list. Furthermore, for operational convenience, a node is appended before the first node of the singly linked list, called ahead node。

  • The data field of the head node can be left empty or can store information such as the list length.

  • Introducing a head node offers two advantages:

    • Since the location of the first data node is stored in the pointer field of the head node, operations at the first position of the linked list are identical to those at other positions, eliminating the need for special handling.
    • Regardless of whether the linked list is empty, its head pointer is a non-null pointer pointing to the head node (the pointer field of the head node is NULL in an empty list), thus unifying the handling of empty and non-empty lists.

Shared stack

Utilizing the characteristic that the bottom position of a stack remains relatively unchanged, two sequential stacks can share a one-dimensional array space. The bottoms of the two stacks are set at the two ends of the shared space, and the tops of the two stacks extend towards the middle of the shared space.

The shared stack is designed to utilize address space more efficiently. Stack overflow occurs if and only if the two stacks are adjacent.

Priority queue (heap)

A heap is a one-dimensional array maintained using the structure of a complete binary tree.

Heaps can be divided intoMax Heapandmin-heap。

  • Max Heap: the value of each node is greater than or equal to the values of its left and right child nodes.

  • min-heap: the value of each node is less than or equal to the values of its left and right child nodes.

Taking a max-heap as an example:

The construction process of a max-heap is to adjust from bottom to top starting from the last non-leaf node.

If the sequence to be sorted is represented by an array, the position of the last non-leaf node is:Array length / 2 - 1

Compare the value of the current node with the value of the left subtree. If the current node is less than the value of the left subtree, swap the current node and the left subtree; after swapping, check whether the left subtree satisfies the max-heap property, and if not, readjust the subtree structure;

Then compare the value of the current node with the value of the right subtree. If the current node is less than the value of the right subtree, swap the current node and the right subtree; after swapping, check whether the right subtree satisfies the max-heap property, and if not, readjust the subtree structure;

When no swap or adjustment is needed, the construction of the max-heap is complete.

Sparse matrix

The number of non-zero elements t in a matrix is very small compared to the total number of elements s, i.e.,s≫ts \gg ts≫ta matrix is calledsparse matrix。

If conventional methods are used to store a sparse matrix, it wastes a lot of space, so only non-zero elements are stored. However, the distribution of non-zero elements is usually very irregular, so the row and column of non-zero elements must also be stored.

Therefore, the non-zero elements and their corresponding rows and columns form a triplet

(行标,列标,值)(\text 行标, 列标, 值) (行标,列标,值)

and then store this triplet according to a certain rule.After compressed storage, the sparse matrix loses the property of random access。

The triplet of a sparse matrix can be stored using eitheran array,ora cross linked list (orthogonal list).

Triplet Sequential List (Array Storage)

  • Store the non-zero elements in a sparse matrix as triplets in row-major or column-major order(row index, column index, value)。
  • Common orders:
    • Row-major: stored from top to bottom by row, and from left to right within each row.
    • Column-major: stored from left to right by column, and from top to bottom within each column.
  • Suitable for situations where the non-zero elements of the matrix are not frequently modified.

Cross Linked List Storage (Cross Linked List / Orthogonal List)

  • More suitable for dynamically modified sparse matrices.
  • Each non-zero element node contains:
    • Value
    • row index
    • Column index
    • Row pointer (pointing to the next non-zero element in the same row)
    • Column pointer (pointing to the next non-zero element in the same column)
  • Enables fast traversal of rows and columns, suitable for operations like matrix multiplication and transposition.

Hash table (Hash Table) also known as a hash list is a data structure, which uses key-value pairs (Key-Value Pair) to store data, and****maps keys to specific positions in an array through a hash function**, thereby achieving efficient search, insertion, and deletion operations.

Concept

  • Storage structure: A hash table usually consists of an array and a hash function. Each element of the array is called a bucket, which can store one or more key-value pairs

  • Hash Function : A hash function converts a key into an index of an array (i.e., an integer)

  • Hashing: is the process of converting input data (usually of arbitrary length) into a fixed-length output (an index within the array range) through a hash function.

  • Hash Collision : A hash collision occurs when different keys are mapped to the same index by the hash function. Common methods to resolve hash collisions include chaining and open addressing.

  • Load Factor：The ratio of the number of elements stored in the hash table to the size of the array (total number of slots).

    • The size of the hash table (number of slots) is fixed, but the number of stored elements can exceed the number of slots.
    • The load factor represents the space utilization efficiency of the hash table.A higher load factor means more elements are stored in the table, indicating high space utilization.
    • WhenWhen the load factor increases (approaches 1 or higher), the lookup efficiency of the hash table may decrease., especially when using open addressing, where conflict resolution requires more probes.
    • When using chaining, an increase in the load factor leads to longer linked lists, thereby increasing the average lookup time.
    • In a dynamic hash table, when the load factor exceeds a certain threshold (e.g., 0.75), it usually triggers the resizing of the hash table to reduce collisions and improve performance.
    • If the load factor is too low, it may trigger downsizing to save space.
    • Ideal load factor:
      • Open addressing: The ideal load factor is usually below 1, recommended between 0.5 and 0.75 to balance space utilization and lookup efficiency.
      • Chaining: The load factor can exceed 1 because multiple elements can be stored in the same slot. Usually, a load factor between 0.7 and 1.0 is a good choice, but it can also be higher, depending on the impact of linked list length on performance.

  • Rehashing: When the load factor of a hash table exceeds a set value, rehashing may be required. Rehashing usually involves creating a larger array and recalculating the hash values of all keys to map them into the new array.

  • Downsizing: Downsizing refers to reducing the capacity of the hash table when the amount of data decreases to a certain level, to save space and improve memory utilization efficiency.

Hash function

A hash function converts a key into an index of an array (i.e., an integer). A good hash function should have the following advantages:

  • Fast computation: The advantage of a hash table lies in its efficiency, so quickly obtaining the corresponding hashCode is very important.

  • even distribution: In a hash table, whether using chaining or open addressing, when multiple elements map to the same position, it affects efficiency. A good hash function should map elements to different positions as much as possible, allowing elements to be evenly distributed in the hash table

Types of keys

Ultimately, a hash function converts keyskeyinto index values. The types that can serve as keyskeyare numerous, such as numbers, strings, objects, etc. Below, we mainly discuss strings as keys, becausethe keys of a hash table are usually implemented as strings, mainly for the following reasons：

  • Consistency: Using strings as keys provides a consistent way to handle different types of keys. Even numbers and other types are eventually converted to strings for storage

  • Hash function: Hash tables use hash functions to map keys to array indices. Strings are usually easier to handle because they can be directly hashed, while other types (like objects) may require more processing steps

  • Ease of comparison: Strings have clear comparison rules, which makes lookup, insertion, and deletion operations simpler and more efficient

  • Space efficiency: In many languages, the storage method of strings as keys is usually more compact than objects or other complex types, reducing memory usage

  • Flexibility: Strings can represent various information and can easily adapt to different scenarios, such as database indexes, field names, etc.

  • Method 1:Add the ASCII values of the characters, the problem is that many keys may end up with the same value, for examplewas/tin/give/tend/moan/tickare all 43

uniform distribution

When designing a hash table, the situation of mapping to the same index value has been handled: chaining or open addressing, but regardless of the scheme, it is for improving efficiency. The best case is still to have data evenly distributed in the hash table, thereforeit is necessary to use prime numbers as much as possible where constants are used, such as the length of the hash table and the base of the Nth power.

Why does using prime numbers make the hash table distribution more uniform?

Because the result of multiplying a prime number with other numbers is more likely to produce unique results compared to other numbers, reducing hash collisions.

For example, the base of the Nth power in Java’s HashMap is chosen as 31. The more commonly used numbers are 31 or 37, which are derived from long-term observation of distribution results.

A hash collision occurs when different keys are mapped to the same index by the hash function. Common ways to resolve hash collisions include chaining and open addressing.

Chaining

Chaining is a relatively common solution to resolve conflicts (also known as the separate chaining method), where each slot (bucket) of the hash table can store multiple elements. When multiple keys are hashed to the same index, chaining stores these elements in a linked list or other data structure.

  • Principle

    • Chaining’score idea is to treat each array element as a bucket, storing multiple key-value pairs within the bucket. Usually implemented using linked lists or arrays

    • Once a duplicate is found, insert the duplicate element intothe beginning or end of the linked listto complete the process

    • When querying, first find the corresponding position based on the hashed index value, thenretrieve the linked list, and search sequentially for the data being looked for

  • Efficiency

    • When implementing a hash table using chaining, the process of finding an element is divided into two parts:
      • Calculate the hash value and locate the slot: This step is a constant time operation, usually 1 operation, independent of the length of the linked list
      • Traverse the linked list to find the element: If multiple elements are on the same slot, the hash table will search for the element in the linked list of this slot. The average number of searches is half the length of the linked list, usually loadFactor / 2
      • Therefore, a successful search may only require looking up half of the linked list: 1 + loadFactor/2, while an unsuccessful search may require traversing the entire linked list to know it’s unsuccessful: 1 + loadFactor
    • Chaining is relatively more efficient than open addressing, so it is used more frequently in real-world development. It does not experience a sharp performance drop after adding an element; for instance, Java’s HashMap uses chaining.

Open addressing

Of open addressingThe main working method is that when a hash collision occurs, it looks for the next empty slot in the array to store the colliding element., there are three ways to probe for an empty slot:

  • Linear probing

    • Linear probing (Linear Probing) checks the next empty slot each time a collision occursindex = (hashFunc(key) + i) % tableSizewhere i is the number of collisions
    • Efficiency

    • Problem: A relatively serious issue with linear probing is clustering

      • For example, when there is no data, inserting 22-23-24-25-26 means that the positions with index values 0-1-2-3-4 all have elements. This continuous block of filled cells is called clustering.

      • Clustering affects the performance of the hash table, impacting insertion, query, and deletion. For example, if we insert a 38, we will find that the continuous cells do not allow data to be placed, and multiple probes are required in this process.

  • Quadratic probing

    • Quadratic probing (Quadratic Probing) determines the position of the next slot based on the square of the number of collisionsindex = (hashFunc(key) + i^2) % tableSize
    • Optimizing the linear problem：
      • Linear probing can be seen as having a step size of1probing, for example, starting from index valuex, then linear probing isx+1，x+2，x+3probing sequentially
      • Quadratic probing optimizes the step size, for example, starting from index valuexStartx+1²，x+2²，x+3², this allows probing a longer distance at once, avoiding the effects of clustering
    • Efficiency：

    • Problem：
      • Quadratic probing still has problems, for example, if what is continuously inserted is32-112-82-2-192, then the step size is the same when accumulated sequentially
      • which means in this caseit will cause a type of clustering with different step sizes, which will still affect efficiency

  • Double Hashing

    • Double Hashing (Double Hashing) uses a second hash function to calculate the step size, determining the position of the next slotindex = (hashFunc1(key) + i \* (constant - (hashFunc2(key) % constant))) % tableSize
    • Optimizing quadratic probing problems:
      • The step size of the probing sequence generated by the quadratic probing algorithm is fixed: 1-4-9-16 and so on, while double hashing uses another hash function on the key, hashes it again, and uses the result of this hashing as the step size
      • The second hashing needs to have the following characteristics:
        • Different from the first hash function (do not use the previous hash function again, otherwise the result will still be the original position)
        • Cannot output 0 (otherwise there will be no step size, every probe will be stepping in place, and the algorithm will enter an infinite loop)
        • Computer experts have designed a well-working hash function:stepSize = constant - (hanshFunc(key) % constant), where constant is a prime number and is less than the capacity of the array
    • Efficiency

Expansion/Shrinking

  • Expansion: As the amount of data increases, the bucket corresponding to each index will become longer and longer, which causes a decrease in efficiency. Therefore, expand the array under appropriate circumstances, such as doubling the capacity

Expansion can simply double the capacity, but in this case, all data items must be modified simultaneously (re-call the hash function to get a different position). For example, a data item with hashCode = 12 is placed at index = 4 when arraySize = 8, and at index = 12 when arraySize = 16

  • Shrinking: When the load factor of a hash table (the ratio of the number of elements to the capacity of the hash table) is lower than a certain threshold, shrinking is considered to save space and improve memory utilization efficiency

When shrinking, the hash table capacity is usually reduced to half of the current capacity, and the positions of all existing elements are recalculated and redistributed

Under what circumstances does expansion/shrinking occur?

A relatively common situation isloadFactor > 0.75expansion is performed when, for example, Java’s hash table expands when the load factor is greater than 0.75

Usually, a lower limit load factor is set, such as 0.25, and shrinking is triggered when it falls below this value, but over-shrinking should also be avoided: usually a smaller, reasonable minimum capacity is set, such as 7 or 8, to ensure that the hash table maintains a certain capacity even when there are very few elements, thereby avoiding frequent expansion and shrinking

The essence of binary search is:Finding the position of the first element in a sequence that satisfies a certain condition。

The confusing part in binary search is usually: when to write less than or equal to in the while loop, and when not to write equal to;

First consider elements sorted in ascending order (descending order is equivalent), which should be divided into two cases:

  1. No duplicate elements;

The problem in this case is generally: find whether a certain element target appears in the sequence, if it appears return the index where it appears, if it does not appear, return the position where the element should be inserted.

That is, determine an interval [x, x], target is the element at position x, if this element does not exist, then the determined interval is [x+1, x], the left boundary is greater than the right boundary, meaning there is no element in the interval.

  1. There are duplicate elements. The latter is a generalization of the former, meaning the algorithm for the latter also applies to the former.

The problem in this case is to determine two sub-problems: the index of the first element greater than or equal to the target, and the index of the first element greater than the target.

(1) Find the lower bound (the index of the first element greater than or equal to the target, x of [x, y))

(2) Find the upper bound (the index of the first element greater than the target, y of [x, y))
