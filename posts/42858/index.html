This article introduces the complete setup process of self-hosting a RustDesk remote desktop server based on Ubuntu and Docker in a local area network environment. It details Docker environment preparation, port settings, and docker-compose deployment, and summarizes the installation of the Windows client, configuration of ID/relay server IP and key, and the final connection method, providing users with a free, high-frame-rate, high-resolution remote control alternative.

After using ToDesk for a few days without shutting down, I used up all the quota in the middle of the month and was not allowed to continue using it for free. I uninstalled it and switched to RustDesk, using self-host. It’s free and the frame rate and resolution are much higher than the free version of ToDesk. Below is the setup record.

Basic Environment Setup

I am using two Windows (Win10) machines and one Ubuntu (configuration as shown below). Using Ubuntu as the ID/relay server, one Windows connects to another Windows. Ubuntu needs Docker installed, and both Windows machines need RustDesk installed and configured.

Note: My three machines are on the same local network.

RustDesk installation and configuration

Installation visit the following link:

RustDeskInstallation

https://github.com/rustdesk/rustdesk/releases

After installation, open network settings, fill in the IP of the ID/relay server (the IP of the server running Docker) into the ID server and relay server fields. The controlled end may not need to fill in the relay server, but the controlling end is recommended to fill it in. Then the controlling end fills in the key (the content of the .pub file under the data folder above).

Finally, enter the ID and password of the controlled end to connect.
