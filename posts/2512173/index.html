GPIO Introduction

GPIO = General-Purpose Input/Output(General-Purpose Input/Output), is a general-purpose pin that can be dynamically configured and controlled during software runtime.

All GPIOs are in input mode by default after power-up. They can be set to pull-up or pull-down via software, can also input interrupt signals, and their drive strength is programmable.。

GPIO Pin Distribution

The RK3568 has 5 groups of GPIOs: GPIO0 to GPIO4. Each group is numbered from A0 to A7, B0 to B7, C0 to C7, and D0 to D7 for distinction. Therefore, the RK3568 theoretically should have 5×4×8=160 5 \times 4 \times 8 = 160 5×4×8=160 GPIOs, but in reality, there are only 152 GPIOs in the datasheet.

This is because the RK3568 actually has a total of 152 GPIOs, among whichGPIO0_D2，GPIO0_D7，GPIO2_C7，GPIO4_D3~GPIO4_D7do not exist, so there are 152 GPIOs.

GPIO Electrical Properties

We take the RK3568 as an example, subject to the specific CPU’s datasheet. The GPIOs on the RK3568 can be set to 3.3V or 1.8V. In actual programming, a high level (3.3V or 1.8V) is represented by 1, and a low level is represented by 0.

How to determine whether the GPIO level of the RK3568 is 3.3V or 1.8V?

Look at the core board schematic to find the GPIO corresponding to the pin and the power domain the pin is connected to, such asMIPI_CAM0_PDN_L_GPIO3_D5

It can be seen that what is connected to this GPIO is VCCIO6

GPIO Electrical Characteristics

The RK3568 TRM manual mentions

GPIO is a programmable GPIO. In addition to IO level, GPIO also has drive strength, pull-up, and pull-down. These concepts are explained as follows:

  • Drive Strength: The drive strength of a GPIO determinesthe output current it can provide. Through software configuration, you can select an appropriate drive strength to ensure that the GPIO can drive the connected external devices or circuits.

  • Pull-up and Pull-down: GPIO pins can determine their default level state through pull-up or pull-down resistors. Through software configuration, you can choose to enable pull-up or pull-down resistors to ensure GPIO maintains a stable default state when no external device is connected。

  • Interrupt: Through software configuration, you can enable the GPIO interrupt function so thatyou can be promptly notified when the GPIO state changes. This is very useful for implementing event-driven applications, where GPIO-triggered events can be handled through interrupts.

  • Multipurpose Pins: Some GPIO pins may have multiple functions, and different functions can be selected through software configuration. For example, a GPIO pin can be configured as a digital input, digital output, PWM output, etc.

IO commands

The io command is a command-line tool for Linux systems, used to read and write values of specified I/O ports. It is mainly used for low-level interaction and debugging with hardware devices, and reading and writing registers at the kernel stage.

The syntax of the command is as follows:

  • Options
    • -b: Perform I/O operations in bytes (default is word).
    • -w: Perform I/O operations in words.
    • -l: Perform I/O operations in double words.
  • Address:Hexadecimal value of the I/O port to read or write
  • Operation:
    • -r: Read the value of the I/O port.
    • -w: Write data to the I/O port
  • Data:Hexadecimal value to write to the I/O port。

Example:

  1. Read the value of the I/O port:
    io -b -r 0x80
    This will read the value of I/O port 0x80 in bytes and display it on the terminal.
  2. Write data to the I/O port:
    io -b -w 0x80 0xAB
    This will write the hexadecimal value 0xAB to I/O port 0x80.
  3. Read in word units:
    io -w -r 0x1000
    This will read the value of I/O port 0x1000 in word units
  4. Write in double-word units:
    io -l -w 0x2000 0xDEADBEEF
    This will write the hexadecimal value 0xDEADBEEF to I/O port 0x2000 in double-word units

LED Pin Register Lookup

The GPIO controlling the LED is GPIO0_B7. We need to configure the GPIO. Generally, we need to configure the GPIO’sMultiplexing register，Direction register，Data registerfor configuration.

Multiplexing register

It can be seen from the RK3568 TRM-Part1 GPIO Interface Description that

The PMU_GRF Register Description is as follows

Therefore, the multiplexing register address = base address + offset address = 0xFDC2000C.

Use the io command to view the address of this register:io -r -4 0xFDC2000C

The register value is 00000001, the [14:12] bits are 000, so the default setting is the GPIO function.

Direction register

It can be seen from the RK3568 TRM-Part1 GPIO Interface Description that the direction register should be GPIO_SWPORT_DDR_L or GPIO_SWPORT_DDR_H

GPIO has four groups of GPIO, namely GPIOA, GPIOB, GPIOC, and GPIOD. Each group is distinguished by numbers A0~A7, B0~B7, C0~C7, D0~D7. GPIO0B7 is in GPIO_SWPORT_On DDR_L, so the offset address of the direction register is 0x0008.

The [31:16] bit attribute is WO, which means write-only. These [31:16] bits are write flag bits, which are the write enable for the lower 16 bits. If a bit in the lower 16 bits is to be set to input or output, the corresponding high bit write flag should also be set to 1.

[15:0] is the lower bits of the data direction control register. If you want to set a GPIO as output, set the corresponding bit to 1; if you want to set a GPIO as input, set the corresponding bit to 0. Then for GPIO0 B7, whether we want to set the 15th bit as input or output, the corresponding [31:16] bit write enable should also be set to 1.

The base address of GPIO0 is 0xFDD60000. Therefore, the address of the direction register = base address + offset address = 0xFDD60000 + 0x0008 = 0xFDD60008

Data register

So the address of the data register is base address + offset address = 0xFDD60000.

The method in the figure above is the same as the method used when analyzing the direction register. As can be seen from the figure above, if you want to control the 15th bit to be high (set to 1), you also need to set the 31st bit to 1. To turn on the light, you need to write 0x8000c040 to the data register.

Summary
  • The base address of the multiplexing register is 0xFDC20000, and the offset address is 000C, so the address to be operated on is base address + offset address = 0xFDC2000C
  • The base address of GPIO is 0xFDD60000, and the offset address is 0x0008, so the address to be operated on for the direction register is base address + offset address = 0xFDD60008. We need to write 0x80000044 to the direction register to set it as output.
  • The base address of GPIO is 0xFDD60000, and the offset address is 0x0000, so the address to be operated on by the data register is base address + offset address = 0xFDD60000
  • Default value of the data register: 0x8000c040 turns on the light, 0x80004040 turns off the light

Controlling GPIO through the mem device and mmap

By opening/dev/memthe device file and mapping it to the memory in user space, we can directly read and write physical memory addresses, thereby achieving control over GPIO registers. This method is more flexible compared to IO commands and allows the use of higher-level programming languages (such as C/C++) to write control logic.

Ways for user mode to access kernel mode in Linux systems

  1. Through read/write/ioctl: Using this method, user-mode programs can communicate with the kernel by reading and writing file descriptors or using the ioctl system call. For example, devices can be controlled or device status can be obtained by reading and writing specific file descriptors.

  2. Through the sysfs virtual file system: sysfs is a virtual file system that represents device and kernel information in the form of files. By reading and writing files under specific paths in sysfs, user-mode programs can interact with the kernel, for example, to control GPIO pins or obtain system information.

  3. Through memory mapping: Memory mapping is a mechanism that maps a memory region in user space to kernel space. Through memory mapping, user-mode programs can directly modify the contents of the memory region to communicate with the kernel. This method can achieve efficient data transmission and sharing.

  4. Via Netlink: Netlink is a communication mechanism provided by the Linux kernel for bidirectional communication between user-mode programs and the kernel. By creating a Netlink socket, user-mode programs can interact with the kernel, send requests, receive event notifications, etc. This method is suitable for scenarios that require complex interaction with the kernel, such as configuring system parameters or sending commands.

/dev/memdevice

/dev/memis a virtual device in Linux systems, usually used in conjunction with mmap, which canmap the physical memory of the device to user spaceto achieve direct access from user space to kernel space. Whether it is a standard Linux system or an embedded Linux system, both support the use of/dev/memdevice.

Direct access to kernel space is a potentially dangerous operation, so only the root user can access the/dev/memdevice. Additionally, some systems may need to separately enable the/dev/memdevice’s functionality.

IO commands are actually based on the/dev/memdevice. If the Linux kernel source code is not configured to support the/dev/mem, IO commands cannot be used.

Use/dev/memThe device requires root permissions, and you mustproceed with caution, because directly accessing physical memory (kernel space) is a potentially dangerous operation that can cause system crashes or data corruption.

Below are the steps to use/dev/memthe basic steps:

Step 1: Open/dev/memthe file

Useopen()the function to open with appropriate permissions and mode/dev/memto get the file descriptor.

  • Access permission options：
    • O_RDONLY: Read-only
    • O_WRONLY: Write-only
    • O_RDWR: Read-write
  • Blocking mode option：
    • Default is blocking
    • O_NDELAYorO_NONBLOCK: Non-blocking

You can choose the appropriate access permission and blocking mode based on actual needs

Step 2: Map physical memory to user space

Usemmap()to map the target physical address to the process’s virtual address space:

  • MMAP_ADDR: Target physical address (usually needs to be page-aligned, e.g., 4KB aligned)
  • MMAP_SIZE: Mapping length (recommended to be at least one memory page, e.g., 4096 bytes)
  • Ifmmap()returnsMAP_FAILED, it indicates mapping failed, check the error code (errno）

Step 3: Read/Write the mapped memory (register operation)

Access hardware registers or physical memory directly via the returned pointer:

  • Pointer type can be adjusted based on register width (e.g.,uint32_t*、volatile uint8_t*etc.)

  • It is recommended to use thevolatilemodifier to prevent access anomalies caused by compiler optimization:

Notes

  1. Permission Requirements: Must be run as root, or have theCAP_SYS_RAWIOcapability.
  2. Address Alignment：mmap()Requires the offset (i.e., physical address) to be an integer multiple of the page size (usually 4096).

GPIO Subsystem API

In the current Linux kernel mainline, there are two versions of the GPIO (General Purpose Input/Output) subsystem, which are distinguished here as the new version and the old version. The new version of the GPIO subsystem interface is implemented based on descriptors (descriptor-based), while the old version of the GPIO subsystem interface is implemented based on integers (integer-based). In the Linux kernel, to maintain backward compatibility, the old version interface is still supported in the latest kernel versions. However, as time goes on, the new version of the GPIO subsystem interface will become more and more refined, eventually completely replacing the old version.

The new GPIO subsystem interface needs to be used in conjunction with the Device Tree. Using the Device Tree and the new GPIO interface allows for more flexible configuration and management of GPIO resources in the system, providing better scalability and portability. Therefore,without the Device Tree, the new GPIO interface cannot be used。

An obvious difference is that the new GPIO subsystem interface uses functions prefixed withgpiod_as the naming convention, while the old GPIO subsystem interface uses functions prefixed withgpio_as the naming convention.
