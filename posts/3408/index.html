Custom Max Heap, TLE

It’s natural to think of using a heap to implement this problem, but a custom heap implementation is required. Although C++'s priority_queue is also a heap, it doesn’t provide a method to modify elements, which greatly limits its use.

Max Heap + Lazy Deletion

We mentioned earlier that priority_queue cannot delete. Using the lazy deletion method, as long as the element that should be deleted is not the maximum value, it won’t be deleted.
After this problem, I finally understand why priority_queue doesn’t provide methods to modify or delete non-top elements.
Lazy Deletion Like 👍

Note that there are two types of “lazy deletion” in this problem: 1. Generated by modification, taskId may appear repeatedly in the heap, so it’s not enough to just check if taskId exists in the map at the end; 2. Generated by deletion, taskId disappears;

Among them, heap.emplace(…) is a member function of the C++ standard library priority_queue, used to directly construct elements in place in the heap, rather than creating an object first and then copying or moving it in. It has similar functionality to push(), but is usually more efficient than push(), especially when constructing complex objects.
