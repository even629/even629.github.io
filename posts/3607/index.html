Problem:

P3607 Power Grid Maintenance

https://leetcode.cn/problems/power-grid-maintenance/description/?envType=daily-question&envId=2025-11-07

DFS + Min-Heap + Lazy Deletion

The code below will WA (Wrong Answer) because it maintains a min-heap of all reachable nodes for each node. If a node is taken offline, it only lazily deletes that node from the heaps of all nodes that can reach it, without considering nodes that can only be reached by passing through this node; these nodes are also unreachable. Moreover: the structure of the power grid is fixed; offline (non-operating) nodes still belong to their respective power grids, and offline operations do not change the connectivity of the grid.

Official Solution:

Disjoint Set Union (DSU)

This method is a bit hard to think of:

Difficulty: Operations are performed online, and offline operations affect subsequent queries.
But the problem states that the grid structure is fixed, meaning going offline does not change connectivity. Therefore, we can think in reverse: if we look at the operations from back to front, ‘going offline’ becomes ‘coming back online’!
So the algorithm adopts a reverse process:

  • First, preprocess the final state of all nodes (which ones are offline at the end).
  • Then process from the last operation backwards.
  • Treat ‘going offline’ as ‘coming back online’.
  • Use Disjoint Set Union (DSU) to maintain the minimum online ID in each power grid.
