Problem:

P224 Basic Calculator

https://leetcode.cn/problems/basic-calculator/description/?envType=study-plan-v2&envId=top-interview-150

First convert to prefix expression, then evaluate the prefix expression

  1. Need to first remove spaces from characters, and for the negative sign (not minus, i.e., unary operator) insert 0 in front of it to make it a binary operator.

    • If ‘-’ is the first character, it is a unary operator (negative sign), insert 0 in front

    • If ‘-’ is preceded by ‘(’, it is a unary operator (negative sign), insert 0 in front

    • In other cases, ‘-’ is a binary operator (minus sign)

  2. Then convert to prefix expression, split operands and operators by spaces. First create a stack opStack to store symbols, let the return value bestring ret, traverse each character of the infix expression.

    • For numbers, continuously add to ret until the next is non-numeric, then add a space after ret to split, indicating this is an operand, convenient for subsequent evaluation of the prefix expression

    • For ‘(’, directly push to opStack

    • For ‘)’, pop all binary operators (‘+’ or ‘-’ or ‘*’ or ‘/’) from opStack and add to ret, until the top of the stack is ‘(’, pop ‘(’ but do not add to ret

    • For binary operators (‘+’ or ‘-’ or ‘*’ or ‘/’)

      • If the top element of the stack is also a binary operator, and the priority of the current binary operator is less than the priority of the top, then keep popping the top until this condition is not met, finally push the current binary operator onto the stack
      • In other cases, push the current binary operator onto the stack.

    • Finally, if opStack is not empty, pop the remaining elements in order and add to ret

  3. Finally evaluate the postfix expression, note that evaluation should preferably use long, then convert to int

Only addition and subtraction

Since the string only contains addition and subtraction operators besides numbers and parentheses, if all parentheses in the expression are expanded, the numbers themselves in the resulting new expression will not change, only the sign in front of each number will change. Parentheses cannot be ignored because although “+” and “-” have the same precedence, parentheses change the precedence.
The main change is the removal of the precedence judgment for binary operators, i.e., the original “if the top element of the stack is also a binary operator, and the precedence of the current binary operator is lower than the top precedence, then keep popping the top of the stack until this condition is not met, and finally push the current binary operator onto the stack” becomes “if the top element of the stack is a binary operator, then keep popping the top of the stack until this condition is not met, and finally push the current binary operator onto the stack”.
