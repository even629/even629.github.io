This article introduces the process of compiling the Topeet RK3568 Linux 5.10 kernel and U-Boot using aarch64-linux-gnu-gcc 15.1.0 in the WSL Archlinux environment. It details the source code modification method to resolve stack pointer errors when configuring clangd, lists various U-Boot defconfig options and their generation results, and points out that the compilation of the root file system and recovery is to be improved later.

Environment

OS

Using Archlinux on WSL

compiler

ARM has released a total of 8 architectures: ARMv1, ARMv2, ARMv3, ARMv4, ARMv5, ARMv6, ARMv7, ARMv8.

Can be used for Cortex-A5x processorsaarch64-linux-gnu-gcc -march=armv8-aCompile code with the command. The ARM GNU compiler can be downloaded from the link below.

Arm GNU Toolchain Downloads

https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads

The version of aarch64-linux-gnu-gcc downloaded via pacman on Archlinux is 15.1.0

View kernel source code with clangd

Generate compile_command.json

When using clangd as a language server, you need to add CompileFlags because clangd does not recognize some GCC compilation options:

SP error

clangd error:

Set the variable current_stack_Bind the pointer to the sp register (i.e., the current stack pointer) to directly read the stack top address. This writing is legal when compiling the Linux kernel with GCC (especially under ARM64 architecture), but Clang has stricter restrictions on this. Therefore we modifyarch/arm64/include/asm/stack_pointer.hThe source code is as follows, for__clang__Use an inline assembly function to return the value of sp

Compile U-Boot

The defconfig for RK3568 is as follows

defconfigSupport kernel dtbDescription
rk3568_defconfigYGeneral version
rk3568-dfu.configYSupport dfu
rk3568-nand.configYSupport MLC/TLC/eMMC
rk3568-spl-spi-nand_defconfigYSPI-nand dedicated SPL
rk3568-aarch32.configYSupport aarch32 mode
rk3568-usbplug.configYSupport usbplug mode

Generate:

  • u-boot（ELF）
  • u-boot.bin(Raw binary)
  • spl/u-boot-spl.bin(If SPL enabled)
  • tpl/u-boot-tpl.bin(If TPL enabled)
  • u-boot.dtb(Embedded external device tree)

The loader generated this way may have some issues, still to be studied TODO
