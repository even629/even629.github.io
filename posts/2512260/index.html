This article describes the process of compiling the Topeet RK3568 Linux 5.10 kernel and U-Boot using aarch64-linux-gnu-gcc 15.1.0 in a WSL Archlinux environment. It details the source code modifications to resolve stack pointer errors when configuring clangd, lists various U-Boot defconfig options and their build results, and notes that the compilation of the root filesystem and recovery is to be completed later.

Environment

OS

Using WSL’s Archlinux

compiler

ARM has released a total of 8 architectures: ARMv1, ARMv2, ARMv3, ARMv4, ARMv5, ARMv6, ARMv7, ARMv8.

For Cortex-A5x processors, you can useaarch64-linux-gnu-gcc -march=armv8-aCommand to compile code, the ARM GNU compiler can be downloaded from the link below

Arm GNU Toolchain Downloads

https://developer.arm.com/downloads/-/arm-gnu-toolchain-downloads

The version of aarch64-linux-gnu-gcc downloaded via pacman on Archlinux is 15.1.0

source code

Compile Kernel

compile

Use the aarch64-linux-gnu-gcc 15.1.0 compiler

clangd to view kernel source code

Generate compile_command.json

When using clangd as the language server, add CompileFlags because clangd does not recognize some GCC compilation options:

sp error

clangd error:

Bind the variable current_stack_pointer to the sp register (i.e., the current stack pointer) to directly read the stack top address. This syntax is legal when compiling the Linux kernel with GCC (especially under ARM64 architecture), but Clang has stricter restrictions on it. Therefore, we modifyarch/arm64/include/asm/stack_pointer.hthe source code as follows, for__clang__Use an inline assembly function to return the value of sp

Compile U-Boot

The defconfig for RK3568 is as follows

defconfigSupports kernel dtbDescription
rk3568_defconfigYGeneral version
rk3568-dfu.configYSupports dfu
rk3568-nand.configYSupports MLC/TLC/eMMC
rk3568-spl-spi-nand_defconfigYSPI-nand dedicated SPL
rk3568-aarch32.configYSupports aarch32 mode
rk3568-usbplug.configYSupports usbplug mode

Generate:

  • u-boot（ELF）
  • u-boot.bin(raw binary)
  • spl/u-boot-spl.bin(if SPL is enabled)
  • tpl/u-boot-tpl.bin(if TPL is enabled)
  • u-boot.dtb(embedded external device tree)
