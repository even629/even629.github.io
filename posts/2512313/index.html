UART

A serial port (Serial Port) is also known as a serial communication interface, and is usually called a COM interface, which is a type of interface for data communication between a computer and external devices (such as serial communication devices)an asynchronous full-duplex interface. It usesserial transmission, meaning it sends only one bit at a time for data transmission.

Specifically, typical serial communication only requires 3 wires: ground (GND), transmit (TX), and receive (RX), as shown in the figure below. Sending and receiving each use a separate wire,No clock line。

Baud rate

Baud rate is a key parameter in serial communication, referring to the number of bits or symbols transmitted per second.

In a digital channel, a pulse signal is a symbol, as shown in the figure below. Symbol rate indicates how many symbols or pulse signals can be sent in 1 second.

Both communicating parties must set the same baud rate to ensure correct data transmission. Common standard baud rates such as 9600, 115200, etc., usually meet the needs of most applications.

However, in specific cases, it may be necessary to set a non-standard baud rate, in which case it must be ensured that all communication devices can support and correctly configure this baud rate.

For setting baud rates higher than 1.5Mbps, it may need to be achieved through fractional or integer division of the clock.

If the required baud rate cannot be achieved through frequency division, it may be necessary to adjust the PLL (Phase-Locked Loop) settings. Adjusting the PLL carries certain risks because it may affect other modules of the device.

Bit raterefers tothe number of bits transmitted per unit of time, usually expressed in bps (bits per second), with the unit being bit/s. Compared to this,Baud rateisthe number of symbols or pulse signals transmitted per second. The relationship between the two can be expressed by the formulaBit rate = Baud rate * log2(M), where M represents the amount of information carried by each symbol.

A symbol is actually a pulse signal, which may carry 1 bit, 2 bits, or more bits of data, depending on the specific implementation of the communication system.In a binary system, the bit rate equals the baud rate, because each symbol carries exactly 1 bit of information。

Example: If the baud rate of the serial port is 9600, then in a binary system, how many bytes can be transmitted in one second?

One byte equals 8 bits, which is 8 high-low level changes, because in a binary system, the bit rate equals the baud rate. Therefore, the number of bytes that can be transmitted in one second is 9600/8=1200 bytes.

Serial communication protocol

Data stream structure

In serial communication, besides paying attention to the baud rate, the structure of the data stream is also crucial.

Each frame of data includes 11 bits:

  • 1 start bit
  • 8 data bits
  • 1-bit parity bit
  • 1-bit stop bit

  • Start bit: Indicates the start of data transmission. The idle state on the data line is 1. Pulling it low from the high level (idle state) indicates the start of data transmission.

  • Data bit: Refers to the number of data bits in each byte, usually 7 or 8 bits.

  • Parity bit: Used to verify the accuracy of transmitted data. Its types include odd parity, even parity, 0 parity, and 1 parity.

  • Odd parity: When the number of 1s in the actual data is even, the parity bit is 1; otherwise, the parity bit is 0.

  • Even parity: When the number of 1s in the actual data is even, the parity bit is 0; otherwise, the parity bit is 1.

  • 0 parity (space parity): The parity bit is always 0; if it is 1, it indicates an error.

  • 1 Parity (mark parity): The parity bit is always 1; if it is 0, it indicates an error.

  • Stop bit: Refers to the number of bits sent after the transmission of each data byte, usually 1 or 2 bits.

Timing waveform analysis

Whenthe data line is at a high level when idle, and duringdata transmission it is pulled low, the first pulse in the figure corresponds to the start bit. It is followed by 8 data bits, which are transmitted in least significant bit (LSB) first order.

For example, the data bits “00110001” convert to hexadecimal 0x31, which is 49 in decimal. In ASCII, the character corresponding to 49 is ‘1’, so the transmitted data is 1. After the data transmission is complete, the bus is pulled high.

Types of serial communication interfaces

UART only specifies the timing of transmission and reception, i.e., “start bit first, then data bits, parity bit, and finally stop bit”. It only specifies high and low levels, but does not specify how many volts a high level or a low level refers to.

Common interface levels for serial ports include TTL，RS232,RS485,RS422, andEach interface usually requires a corresponding level conversion chip. DirectlyWhen using the serial port interface brought out by the processor directly, it is usually TTL level。

However, there may be level differences between different models or suppliers of processors, which means that in some cases devices cannot be directly connected. Therefore, in order to ensure normal communication, appropriate level conversion must be performed. The general serial port interface level comparison is shown in the table below:

RS232 interface

RS232（Recommended Standard 232） The protocol is a serial communication standard established by the Electronic Industries Association in 1970. This standardunified the connectors and pin definitions for serial communication, as shown in the figure below, and clearly specifies the level standard for each connector pin

DB9 pin description

DB9 is a common serial port connector, typically used for RS-232 serial communication and other serial communication applications. It contains 9 pins, each with a specific function. Below is the pin description of the DB9 connector:

  • Pin 1 - DCD (Data Carrier Detect):Data Carrier Detect. Indicates whether the remote device is ready for communication.
  • Pin 2 - RXD (Receive Data):Receive Data. Receives data stream from the remote device.
  • Pin 3 - TXD (Transmit Data):Transmit Data. Sends data stream to the remote device.
  • Pin 4 - DTR (Data Terminal Ready):Data Terminal Ready. Indicates that the data terminal equipment (such as a computer) is ready for communication.
  • Pin 5 - GND (Ground):Ground. Electrical ground, used for circuit reference and shielding.
  • Pin 6 - DSR (Data Set Ready):Data Set Ready. Indicates that the remote device is ready to receive and send data.
  • Pin 7 - RTS (Request to Send):Request to Send. The sender uses this signal to request to start sending data.
  • Pin 8 - CTS (Clear to Send):Clear to Send. The receiver uses this signal to indicate that it is ready to receive data.
  • Pin 9 - RI (Ring Indicator):Ring Indicator. Indicates that the remote device has sent a ring signal.

RS232 is essentially also a serial port protocol, the same as the serial port protocol. However, it specifies the physical interface and level characteristics of the serial port, so it differs at the hardware level, but there is no difference in software programming and the implementation of serial communication.

Level characteristics

  • The electrical signal at the RS232 receive-transmit end isrelative to the common ground（GND）voltage signal。

    • In the RS232 standard,A voltage differential between +3V and +15V is defined as logic “0”, while between -3V and -15V represents logic “1” A voltage differential between -3V and +3V is undefined.
    • Typically, in practical applications, it is desirable for the absolute value of the voltage differential to be between 5V and 15V to ensure reliable signal transmission.
    • When transmitting data, the transmitter driver outputs a positive voltage signal of +5V to +15V to represent logic “0”, while a negative voltage signal of -5V to -15V represents logic “1”.
    • When receiving data, any electrical signal detected above 3V is considered a valid signal.

  • The RS232 interface has relatively high current capability during transmission and reception, allowing it to handle relatively large current loads, making it suitable for long-distance communication and connecting external devices.

  • RS232 signals have high anti-interference capability, enabling stable operation in industrial environments and resistance to electromagnetic interference.

  • RS232 includes the following signal lines:

    • Transmit line (Tx)
    • Receive line (Rx)
    • Ground line (Ground)
    • Data Terminal Ready (DTR)
    • Data Set Ready (DSR)
    • Request to Send (RTS)
    • Clear to Send (CTS)
    • Data Set Ready (DSR)

  • RS232 supports various baud rates, typically from low to high speeds, reaching up to hundreds of kilobits per second (kbps).

RS485 interface

The RS485 standard was jointly developed by the Telecommunications Industry Association (TIA) and the Electronic Industries Alliance (EIA). The main purpose of this standard isto address long-distance communication needs（up to 1200 meters）and to provide excellent anti-interference performance。

Features and Advantages

  • Long-distance communication capability: RS485 can achieve a communication distance of up to 1200 meters under ideal conditions, making it suitable for applications requiring long-distance data transmission.
  • Strong anti-interference capability: Due to the adoption of differential signal transmission, RS485 can effectively resist electromagnetic interference (EMI) and radio frequency interference (RFI), ensuring the stability and reliability of data transmission.
  • Multi-station capability: RS485 supports communication among multiple devices (up to 32) on the same bus, where each device can independently send and receive data, thereby achieving flexible network networking.
  • Wide application: RS485 is widely used in industrial automation control systems, building automation, smart home systems, power system monitoring, and other fields, meeting the requirements for long-distance, high-speed, and reliable communication

Level characteristics

The level characteristics of RS485 are as follows:

  • Differential signal transmission: RS485 uses differential signals for data transmission, meaning the data signal is represented by the voltage difference between two signal lines (usually marked as A and B lines). This differential signal transmission method gives RS485 good anti-interference capability and long-distance transmission capability.
  • Voltage range：
    • Logic ‘1’ is represented by a voltage difference of +(2~6)V between the two lines；
    • Logic ‘0’ is represented by a voltage difference of -(2~6)V between the two lines。
    • The interface signal level compared to RS232 is reduced, making it less likely to damage the interface circuit chip, and this level is compatible with TTL Level compatible , allowing easy connection to TTL circuits.
  • Current capability: The RS485 transmitter has a strong driving capability, capable of driving long communication lines and multiple receivers. The receiver can handle large input currents to ensure reliable signal reception.
  • Electrical characteristics: RS485 supportsmultiple devices（up to 32 a）communicating on the same bus. It supports a communication distance of up to 1200 meters, and even longer under specific circumstances

In simple terms:

  • REWhen at a low level, RO output is valid. Conversely, RO is in a high-impedance state, meaning it is invalid. RO is connected to UART7_RX_M1，UART7_RX_M1 is the serial port's receive pin. Therefore, when /RE is at a low level, the serial port can receive data; conversely, the serial port cannot receive data.
  • When DE is high, DI is active, and DI is connected to UART7._TX_M1，UART7_RX_M1 is the serial transmit pin. Therefore, when DE is high, a low level on DI causes the driver’s non-inverting terminal A to output a low level, and the driver’s inverting terminal B to output a high level, allowing the serial port to send data; otherwise, the serial port cannot send data.

RE and its active level is exactly opposite to DE., and because RS485 ishalf-duplex, it cannot transmit and receive simultaneously. Therefore,RE and DE definitely cannot be enabled at the same time,meaningRE the levels of and DE must be the same. In this way,REenabling is equivalent to disabling DE, so these two pins are connected together.

This makes it very clear. If GPIO0_C6 outputs a low level,RE and DE are at a low level,RE when is at a low level, the serial port can receive data; when DE is at a low level, the serial port cannot send data.

If GPIO0_C6 outputs a high level,REand DE are high,RE When it is high, the serial port cannot receive data; when DE is high, the serial port can send data.

That is to say, we need to use GPIO0_the high and low levels of the C6 pin to control whether the RS485 transmits or receives. Therefore, we need to write a driver to achieve this. The source code provided by Xunfor configures the driver to control GPIO0 by default_C6。

auto-transmit/receive 485 circuit

In addition to software control, automatic switching can also be implemented through hardware to realize the automatic transceiver function of RS485. There is a certain time difference in software-controlled transmission and reception. To reduce this time difference, the baseboard schematic of the Xunfor development board has been optimized to be compatible with hardware-based automatic transceiving. The specific modification method is to remove R295 and solder all components marked as DNP in the figure below.

  • When the UART_TX_M1 is high, the base of Q17 is also high, causing Q17 to conduct,RE and DE are low, so the serial port is in receive data mode
  • When the UART_TX_M1 is low, the base of Q17 is low, causing Q17 to cut off,RE and DE are high, the serial port is in transmit data mode. Since the UART_TX_M1 is high when idle, indicating that the serial port is in the receive data state; when the UART_TX_M1 is pulled low, the serial port is in the transmit data state.

Since the UART_TX_M1 high level indicates receive mode, so when sending a 1, it is also a high level. This means the chip will always be in receive mode, causing the 1 to fail to be sent out, right?

When the adapter chip is in receive mode, the A and B pins will be in a high-impedance state。

High impedance means a very large resistance, almost equivalent to an open circuit.At this time, A is pulled high by the pull-up resistor, and B is pulled low by the pull-down resistor。

Therefore,A high and B low represent a 1 in communication. Through this ‘receive mode’, we cleverly send out the ‘1’.

This optimization saves an RS485 transceiver control IO, allowing the RS485 to be used completely as a serial port, which makes writing drivers easier.

Serial port subsystem framework

The serial port subsystem framework is a modular framework in the Linux kernel specifically used for handling serial port devices, as shown in the framework diagram below.

  • Application layer: Located at the topmost layer, it is the interface between user-space applications and kernel space in the serial port subsystem. The application layer includes user-space serial port applications, such as the serial communication tool minicom.
  • Character device layer: Located below the application layer, it is responsible for passing serial port read/write requests from user space to the tty_core layer in kernel space. The character device layer treats the serial port device as a special character device and operates it through the character device interface.
  • tty_core layer: Located below the character device layer, it is the core module in the Linux kernel used to manage serial port devices. It handles the basic functions of serial port devices, such as data transmission, control, and buffer management. The tty_core layer is independent of specific serial port hardware and serves as the general processing layer for serial port devices.
  • uart_core layer: Located below the tty_core layer, it provides the low-level driver interface for serial port devices and is responsible for communicating with specific serial port hardware. uart_core layer is responsible for controlling low-level operations such as serial data transmission and reception, interrupt handling, and clock management.
  • Hardware layer: Located at the lowest level, it is the part of the serial port subsystem related to specific hardware. The hardware layer includes the driver programs for serial port hardware, communicates with specific serial port controllers, and implements low-level control and operation of the hardware.

struct termios structure

struct termiosis a structure in the Linux kernel used to describe the parameters of terminal devices (including serial port devices).

It is defined in<linux/termios.h>header file, containing multiple fields for configuring and managing the attributes and behavior of terminal devices, including input/output baud rate, data bits, parity bits, stop bits, etc. Its definition is as follows:

Below are some important fields of the struct termios structure:

  • tcflag_t c_iflag: This field contains input mode flags, used to configure the input behavior of the terminal device, such as input control characters, input data processing, etc.
  • tcflag_t c_oflag: This field contains output mode flags, used to configure the output behavior of the terminal device, such as output data processing, output control characters, etc.
  • tcflag_t c_cflag: This field contains control mode flags, used to configure the control parameters of the terminal device, such as baud rate, data bits, stop bits, parity bits, etc.
  • tcflag_t c_lflag: This field contains local mode flags, used to configure the local operations and input/output behavior of the terminal device.
  • cc_t c_cc[NCCS]: This field contains a special control character array, used to configure the control characters of the terminal device, such as the terminal’s erase character, end character, stop character, etc.

Common serial port control functions

tcgetattr()

Function: Gets the attribute configuration of the current terminal (or serial port device).
Prototype：

  • Parameters：
    • fd: File descriptor (e.g., opened serial port/dev/ttyS0）
    • termios_p: Pointer tostruct termiospointer, used to store the current configuration
  • Return value: Returns 0 on success, -1 on failure
  • Usage: Read the current serial port’s baud rate, data bits, parity, flow control, input/output mode, and other settings.

Usually, this function is called before modifying the serial port to save the original configuration, so that it can be restored when the program exits.

tcsetattr()

Function: Set the attribute configuration of the terminal (or serial port).
Prototype：

  • Parameters：
    • fd: File descriptor
    • optional_actions: Specifies when to apply the new settings, common values:
      • TCSANOW: Take effect immediately
      • TCSADRAIN: Take effect after output is complete (commonly used for output-related settings)
      • TCSAFLUSH: Take effect immediately after flushing the input/output buffers
    • termios_p: Pointer to the new configurationstruct termiosPointer
  • Return value: Returns 0 on success, -1 on failure
  • Usage: Apply new serial port parameters (e.g., baud rate, 8N1 configuration, raw mode, etc.)

⚠️ It is recommended to check the return value after modification to ensure the setting is successful.

cfgetispeed() and cfgetospeed()

Function: Get the input (receive) and output (transmit) baud rates, respectively.
Prototype：

  • Return value: Baud rate constants (e.g.,B9600,B115200), not actual values (e.g., 9600)
  • Note: Returnsspeed_ttypemask value, cannot be used directly as an integer

The actual baud rate must be converted via a lookup table or system-specific method (some systems providecfmakeraw()orioctlto obtain the real rate).

Function: Set the input and output baud rates, respectively.

Prototype：

  • Parameters：

    • termios_p: Points to thetermiosstructure to be modified
    • speed: Baud rate constant (e.g.B9600,B115200）

  • Return value: Returns 0 on success, -1 on failure

  • Typical usage：

Usually the transmit and receive baud rates are set to the same value (full-duplex communication), unless there are special requirements.

Function: Flushes the terminal’s input or output queue.

  • queue_selectorValues:

Introduction to GPS data frames

The following are common data types in the GPS NMEA format, including:

  • GPRMC (Recommended Minimum Specific GPS/Transit data): The GPRMC data type provides the most basic GPS positioning information such as position, speed, and heading. This data type is commonly used in navigation systems and ship automatic navigation systems.
  • GPVTG (Track Made Good and Ground Speed): The GPVTG data type provides heading angle and ground speed information, used to display navigation information for moving objects such as ships and vehicles. It provides the ground heading angle and speed relative to the ground.
  • GPGGA (Global Positioning System Fix Data): The GPGGA data type includes positioning solution, time, position accuracy, altitude, and other location information, usually used for the receiver to display the current position in real-time.
  • GPGSA (GPS DOP and Active Satellites): The GPGSA data type includes DOP values and current satellite positioning status. It is used to provide information such as the number of available satellites in GPS satellite measurement data and the calculated position geometric factor DOP.
  • GPGSV (GPS Satellites in View): The GPGSV data type provides information on currently visible satellites, including satellite PRN, elevation angle, azimuth angle, signal strength, etc. This information helps the receiver find more satellites and improve the accuracy of position positioning.
  • GPGLL (Geographic Position - Latitude/Longitude): The GPGLL data type provides longitude and latitude information, used to describe the current latitude and longitude position of the receiver.
