This article describes how to compile OpenSSL 1.0.0s for aarch64 on x86_64 host using ARM cross toolchain to compile OpenSSL 1.0.0s aarch64 shared library complete process, including environment preparation, Configure configuration (need to use linux-aarch64 target and through environment variable CROSS_COMPILE specify toolchain prefix), modify ARFLAGS and AR definitions in three Makefiles to solve thin archive and parameter order issues, as well as compilation, installation and common problem troubleshooting methods.

Objective: On x86_64 host, useaarch64-none-linux-gnu-cross toolchain, compilelibssl.soandlibcrypto.so(ARM aarch64 shared library).

1. Environment preparation

1.1 Source code

If you don’t have to use openssl-1.0.0, openssl 1.1.1 is more recommended, not only the code is more robust, but also compilation is simpler.

1.2 Cross toolchain

This article uses Arm GNU Toolchain 11.2-2022.02（aarch64-none-linux-gnu-triplet).

ARM has released a total of 8 architectures: ARMv1, ARMv2, ARMv3, ARMv4, ARMv5, ARMv6, ARMv7, ARMv8.

For processors supporting the ARMv8 instruction set, you can use-march=armv8-aparameter to compile code. The ARM GNU compiler can be downloaded from the link below

2. Configuration (Configure)

OpenSSL 1.0.0 does notlinux-aarch64This target (even if the command line can select it, it will fall back to the host’scc, the output is x86_64).
The correct approach is to use the generic 64-bit targetlinux-generic64, and throughenvironment variableCROSS_COMPILEspecify the toolchain prefix.

⚠️ Important：CROSS_COMPILEMust use environment variable pass in.
If written as./Configure ... CROSS_COMPILE=...command line argument,
The Configure script of OpenSSL 1.0.0 will report:
target already defined - linux-generic64 (offending arg: CROSS_COMPILE=...)

Aboutno-xxx(disabled algorithms/protocols)

OpenSSL’s./ConfigureAcceptno-<feature>BeforeRemove at compile timeunnecessary algorithms, effect:

  • Reduce size: disabled algorithms are not compiled into.so, library file is smaller
  • Avoid patents: IDEA, RC5 were patented algorithms (expired, but old versions still need explicit enabling)
  • Eliminate security risks: MD2/SSL2/SSL3 have been proven insecure, no need to keep

Suggestion: If unsure which algorithms are needed,adding all is the safest— can significantly reduce the compiled product size and eliminate known security risk algorithms.

If your project really needs a certain algorithm (e.g., legacy devices that must support SSL3), remove the correspondingno-xxxThat’s it.

About-DOPENSSL_NO_HEARTBEATS(compile macro)

OpenSSL 1.0.0 itself does not support Heartbeat(Heartbeat was introduced in 1.0.1), so this macro on 1.0.0 isredundant.
But it’s harmless to include, and if you switch to 1.0.1+ version in the future, it will automatically prevent Heartbleed.

  1. Thin archive issue: For some versions of cross toolchains (including the Arm GNU Toolchain 11.2 used in this article), thearBeforeARFLAGSwhen empty, by default createsthin archive(thin archive) — the archive only stores the symbol table, does not embed.ofile content. When linking,ldreading members will fail:
  1. AR parameter order issue：arthe syntax isar [-]X[MOPTS] archive [members], the operator (r) must be immediately adjacent toaror placed beforeMOPTSthe preceding. Wrong orderar rc rwill causearto treat the lastras the archive filename.

Fix: changeARFLAGStoc(create), and changeARin the definitionrandARFLAGSSwap order.

⚠️ Important: OpenSSL 1.0.0 has 3 Makefile needs to be modified (top-level,crypto/、ssl/), becausecrypto/Makefileandssl/Makefileeach has its ownARdefinition, andhardcodedar r(no$(CROSS_COMPILE)prefix). The top-level Makefile passesmake -ePassARenvironment variables, but in the sub-MakefileARthe definition has higher priority.

After modification, verify:

Expected output:

Actual call after expansion:aarch64-none-linux-gnu-ar rc ../../libcrypto.a *.o— correct fat archive format.

Principles

  • arMeaning of the mode letters in the command:

    • r: replace/insert member (operation)

    • c: create archive (if not exists) (modifier)

    • s: write symbol index (equivalent to running separatelyranlib）

  • GNU ar syntax:ar [-]X[MOPTS] archive [members]

    • X = operation（r/d/m/p/q/t/x）

    • MOPTS = modifiers（c/v/w/…）

wrongar $(ARFLAGS) r(becomes when ARFLAGS=car c r) will makearPutras archive name. The correct way isar r$(ARFLAGS)→ar rc。
