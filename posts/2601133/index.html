network communication basics

development history

In 1969, ARPAnet initially connected computers at four American universities, using packet switching technology to achieve data transmission between computers.

In 1972, the first transatlantic connection with computers in the UK and Norway was achieved.

In 1973, research began on the Transmission Control Protocol (TCP) and Internet Protocol (IP).

In 1984, the TCP/IP protocol became the standard for computer network communication.

In 1986, the US National Science Foundation (NSF) established a wide area network (the predecessor of the Internet).

In 1994, China’s NCFC network was officially connected to NSFnet, and China joined the Internet.

Network protocol

A network protocol is a set of rules and conventions for communication between devices in a network. Common network protocols include:

TCP/IP protocol

The TCP/IP (Transmission Control Protocol/Internet Protocol) protocol, namely the Transmission Control Protocol/Internet Protocol, is the fundamental protocol suite used for Internet communication.

Characteristics

  • Connection-oriented: Before transmitting data, the TCP protocol establishes a connection through a “three-way handshake” to ensure the reliability of both communicating parties and the orderly nature of data transmission.
  • High reliability: TCP ensures the reliability and integrity of data through mechanisms such as sequence numbers, acknowledgments, retransmission, and flow control. If a data packet is lost or damaged during transmission, TCP will request retransmission until the data arrives accurately.
  • Byte-stream based: TCP treats application-layer data as an unstructured byte stream and does not preserve message boundaries. This indicates that TCP does not care about the specific content of the application-layer data, but merely transmits it as a sequence of bytes.

Function

  • The TCP/IP protocol adopts a layered working method, with each layer assuming specific functions, providing a complete communication solution from network hardware to applications. It enables data exchange between different computers and networks.

UDP protocol

UDP (User Datagram Protocol) is the User Datagram Protocol. Unlike TCP, UDP does not provide a connection-oriented communication method, nor does it ensure the reliability and integrity of data.

Characteristics

  • Connectionless nature: UDP does not need to establish a connection before sending data; each data packet is sent independently, and the receiver does not need to send an acknowledgment. This characteristic gives UDP lower latency and higher transmission efficiency.

  • Unreliability: UDP cannot guarantee the order, integrity, and reliability of data packets. If a data packet is lost or damaged during transmission, UDP will not request retransmission.

  • Datagram-based: UDP encapsulates application-layer data into independent datagrams for transmission, with each datagram containing complete destination and source address information.

Function

UDP is suitable for scenarios with high real-time requirements, such as video streaming, audio streaming, and online games. In these applications, even if a small number of data packets are lost, it will not seriously affect the overall experience.

DNS Protocol

DNS (Domain Name System) is a distributed database system whose main function is to convert domain names into corresponding IP addresses (e.g., 192.0.2.1). With DNS, users can access websites through easy-to-remember domain names without having to memorize complex IP addresses.

DHCP Protocol

DHCP (Dynamic Host Configuration Protocol) is a network protocol whose function is to automatically assign network parameters such as IP addresses, subnet masks, default gateways, and DNS server addresses to client devices within a local area network.

FTP Protocol

FTP (File Transfer Protocol) is a protocol used for network file transfer, which can be used to share files between different computers, and is one of the important tools for file transfer on the internet.

HTTP/HTTPS Protocol

HTTP (HyperText Transfer Protocol) is an application layer protocol used for distributed, collaborative, and hypermedia information systems, and is the foundation of data communication on the World Wide Web (WWW).

HTTPS (HyperText Transfer Protocol Secure) is the secure version of HTTP, which encrypts data through the SSL/TLS protocol to ensure the security of data transmission.

Function

The HTTP protocol enables users to access various resources on the internet through a browser, such as web pages, images, videos, etc. The HTTPS protocol is dedicated to protecting user data security, preventing data from being stolen or tampered with during transmission, and is widely used in scenarios with high data security requirements such as online banking, online payments, and e-commerce.

Transmission Medium

Including network cables, cables, and optical fibers

  • Network Cable

A network cable is generally composed of several wires, commonly 8 copper wires twisted together in pairs to form 4 twisted pairs. The exterior is wrapped with an insulating layer and a sheath, which serves to protect the internal wires, prevent information leakage, and protect against external electromagnetic interference.

  • Cable

A cable is a device for transmitting electrical power or signals, usually composed of several wires or groups of wires. A cable consists of a conductor, an insulating layer, and a sheath.

  • Optical Fiber

Optical fiber is a cable that uses light as the information carrier and utilizes the principle of total internal reflection of light in glass or plastic fibers for data transmission. It is mainly divided into single-mode fiber and multi-mode fiber.

Types of Network Cables

Category 1 and Category 2 cables

Category 1 and Category 2 cables were mainly used in early telephone communication systems and did not support data transmission or modern network applications. With the continuous advancement of network technology, these two types of cables have been phased out and are no longer used for network cabling.

Category 3 cable (Cat 3)

Category 3 cable is one of the early Ethernet cable standards, mainly used for traditional Ethernet with a transmission rate of 10Mbps. In modern network environments, Cat 3 cables are rarely used.

Category 4 cable (Cat 4)

Compared to Cat 3, Cat 4 was an improvement, but in practical applications, due to its low cost-performance ratio, it was not widely adopted and was quickly replaced by the more advanced Cat 5.

Category 5 cable (Cat 5)

5 Category cable is a commonly used twisted-pair cable in computer networks，Supports a maximum 100Mbps data transmission rate（FastEthernet）。

Category 5e cable (Cat 5e)

Enhanced 5 Category cable is 5 an enhanced version of Category cable，mainly used for Gigabit Ethernet（1000Mbps），while backward compatible with Fast Ethernet。

Category 6 cable (Category 6, Cat 6)

Compared to previous cables，6 the cable in terms of internal structure、materials, and external protection have been improved，to ensure higher transmission quality and stability。6 The cable is also used for Gigabit Ethernet（1000Mbps）as the transmission medium

Category 6A cable (Category 6A, Cat 6a)

Super 6 cable is 6 a further upgraded version of the cable，supports 10Gbps the transmission rate of（10 Gigabit Ethernet）。

Category 7 cable (Category 7, Cat 7)

7 Category 7 cable belongs to shielded twisted pair（STP），supports 10Gbps transmission rate of，has higher bandwidth and lower signal attenuation rate。

Network cable wiring standards

There are two standards for making network cables, namely T568A and T568B.

If both ends of the network cable use the same wiring sequence standard, it is called a straight-through cable, used to connect different types of devices, such as a computer and a router;

If the wiring sequences at both ends are different, such as one end being T568A and the other being T568B, it is called a crossover cable, often used to connect the same types of devices, such as computer to computer, or switch to switch.

T568A

T568A standard wiring sequence: Green-White, Green, Orange-White, Blue, Blue-White, Orange, Brown-White, Brown.

T568B

T568B standard wiring sequence: Orange-White, Orange, Green-White, Blue, Blue-White, Green, Brown-White, Brown

Network model

OSI seven-layer model

The OSI seven-layer network model is a network architecture standard formulated by the International Organization for Standardization (ISO) specifically to achieve interconnection and intercommunication between different systems.

The OSI seven-layer model, from the bottom to the top, consists of: Physical Layer, Data Link Layer, Network Layer, Transport Layer, Session Layer, Presentation Layer, and Application Layer, as shown below:

  • Physical Layer

The physical layer only needs to consider how to send 0s and 1s, and how the receiver identifies them to complete the transmission of the bit stream, without needing to care about the specific meaning of the bit stream. In addition, it also specifies some electrical characteristics in the network.

  • Data Link Layer

Simple 0s and 1s are meaningless; the data link layer converts the raw bit stream from the physical layer into individual frames. It is also responsible for the establishment, maintenance, and termination of link connections, and also has functions such as frame synchronization, error control, and flow control.

  • Network Layer

The network layer mainly handles the transmission and routing of data packets from the source node to the destination node. That is, it establishes host-to-host communication.

  • Transport Layer

The transport layer is responsible for providing reliable port-to-port data transmission services. It ensures that data is transmitted accurately from the sender to the receiver.

  • Session Layer

It is mainly responsible for managing user sessions and controlling the dialogue between applications, including the start, end, and data exchange process of the dialogue.

  • Presentation Layer

The presentation layer is responsible for the conversion of data formats, such as the conversion between the device’s inherent format and the network standard format, which can be understood as acting as a “translator”.

  • Application Layer

The application layer is the layer directly facing the user, completing specific network applications through the interaction between applications.

TCP/IP Four-Layer Model

The TCP/IP four-layer model is a simplification of the OSI seven-layer model. It divides network functions into the network interface layer, internet layer, transport layer, and application layer, as shown in the figure below:

  • Application Layer

The application layer directly interacts with applications. Common application layer protocols include HTTP, FTP, SMTP, DNS, etc. Different types of applications will choose different protocols in the application layer according to their needs; for example, mail transfer applications can choose the SMTP protocol.

  • Transport Layer

It provides reliable port-to-port data transmission services. The most important protocols in the transport layer are TCP and UDP. TCP provides connection-oriented, reliable services. UDP provides connectionless, unreliable but fast services.

  • Network Layer

It provides communication between hosts. The most important protocol in the network layer is the IP protocol, which adds an IP address to each data packet.

  • Network Interface Layer

The network interface layer includes the physical layer and the data link layer, so the network interface layer includes both physical transmission and the establishment of links.

OSI (Open System Interconnection) is a seven-layer model developed by the International Organization for Standardization (ISO).

The TCP/IP four-layer model refers to the OSI seven-layer model and simplifies it into four layers, as shown in the figure below. Therefore, it is more efficient and lower in cost in applications. Thus, it rapidly developed and became the de facto standard.

Encapsulation and Decapsulation of Network Data

In network communication, encapsulation refers to the process of adding the protocol header and trailer of the current layer to the data from the upper layer, forming a new data unit, and then passing it to the next layer for processing.

Decapsulation is the reverse process of encapsulation, where the receiver removes the protocol header and trailer of the current layer from the received data unit, extracts the data from the upper layer, and then passes it to the upper layer for processing.

The processes of encapsulation and decapsulation are carried out sequentially at different network layers.

Application Layer Encapsulation: Data generated by applications, such as web page data transmitted by the HTTP protocol, email data transmitted by the SMTP protocol, etc., is referred to as application data at this point. Application layer protocols format the data according to their own requirements; for example, HTTP adds request or response headers, containing information such as request methods, URLs, and response status codes.

Transport Layer Encapsulation: Application data arrives at the transport layer (such as TCP or UDP). Taking TCP as an example, TCP adds a TCP header to the data, containing information such as source port number, destination port number, sequence number, acknowledgment number, and control bits. Port numbers are used to identify applications, enabling the receiver to correctly deliver the data to the corresponding application. The UDP header is relatively simple, also containing source and destination port numbers, used for multiplexing and demultiplexing. At this point, the data with the TCP or UDP header added is called a segment.

Network Layer Encapsulation: The segment from the transport layer arrives at the network layer (such as IPv4 or IPv6), and the network layer adds an IP header. The IP header contains key information such as source IP address and destination IP address, used for routing between different networks. The data with the IP header added is called a packet. For example, when you access a website, your device encapsulates the TCP segment in an IP packet, with the destination IP address being the IP of the website server.

Data Link Layer Encapsulation: The packet from the network layer arrives at the data link layer (such as Ethernet), and the data link layer adds a frame header and trailer. The Ethernet frame header contains source MAC address, destination MAC address, etc., and the trailer contains a Cyclic Redundancy Check (CRC) value for error detection. The data at this point is called a frame. The frame is converted into electrical or optical signals by the physical layer for transmission over the physical medium.

Decapsulation is the reverse process of encapsulation. At the receiving end, data starts from the physical layer and passes upward through each network layer sequentially, with each layer removing the corresponding header information to restore the data into a form that the application layer can process, as shown in the figure below:

Data Link Layer Decapsulation: The physical layer receives electrical or optical signals, converts them into data frames, and passes them to the data link layer. The data link layer checks the frame header and trailer, ensures the data transmission is error-free through CRC check, removes the frame header and trailer, and passes the data packet to the network layer.

Network Layer Decapsulation: The network layer receives the data packet, checks the IP header, and determines whether it is destined for the local machine based on the destination IP address. If so, it removes the IP header and passes the segment to the transport layer.

Transport Layer Decapsulation: The transport layer receives the segment, delivers the data to the corresponding application based on the port number in the TCP or UDP header, and removes the TCP or UDP header. At this point, the data is restored to application layer data, and the application layer program can process this data.

Hardware Connection

RJ45 Interface

RJ45 is Registered Jack 45 the abbreviation for. In computer networks, RJ45 is the abbreviation for the standard 8-pin modular interface. RJ45 consists of a plug and a socket, which are connected by a network cable. The RJ45 plug is shown in the figure below

In appearance,the RJ45 plugis rectangular, with a plastic shell on the outside,there are 8 metal contacts inside, which are used for electrical connection with the RJ45 socket。

Corresponding to the RJ45 plug, RJ45 jackalso has 8 metal contacts for connecting with the RJ45 plug. When the RJ45 plug is inserted into the jack, the 8 metal contacts on the front of the plug make tight contact with the metal contacts inside the jack, thereby enabling signal transmission.

Both 100M and Gigabit network ports use RJ45 interfaces for physical connection. However, although they look exactly the same, their transmission capabilities are quite different. The corresponding pin usage is also quite different.

Differential signal: TX+/- and RX+/- are paired differential signal lines that resist interference through differential transmission, improving the stability of Ethernet data transmission. This is the core design of the RJ45 network port.

Transmission rate and pin utilization: 100M Ethernet (100BASE-TX) only uses 4 pins: 1/2 (TX), 3/6 (RX), whileGigabit Ethernet (1000BASE-T) enables all 8 pins, transmitting and receiving simultaneously through 4 pairs of differential lines to achieve a 1Gbps transmission rate.

Function of bi-directional pins (BI): Pins 4/5 and 7/8 are designed for bi-directional transmission. In Gigabit mode, these two pairs dynamically switch their transmit/receive direction based on communication needs, working with Digital Signal Processing (DSP) technology to achieve full-duplex communication.

Differential transmission characteristics: Each pair of pins (such as TX_D1+/TX_D1-) are differential signal lines, transmitting signals through the voltage difference between the two wires. This effectively suppresses common-mode interference and improves the stability of high-speed data transmission, which is the core design of the Ethernet physical layer.

Wiring standard compatibility: This pin definition still follows the T568A/T568B wiring standard, and the wire sequence arrangement is consistent with Fast Ethernet ports, so Gigabit Ethernet cables can be backward compatible with Fast Ethernet ports.

MAC Controller

The full name of the MAC Controller is Media Access Control. It belongs to OSI Layer 2 (Data Link Layer) a part of.

The main function of the MAC Controller is to control the connection with the PHY chip. It acts like a “bridge manager”, responsible for managing the transmission of data between the Data Link Layer and the Physical Layer, ensuring that data can flow smoothly between the two layers.

The main functions of the MAC Controller include:

  1. Encapsulation and parsing of Ethernet frames
    • Responsible for adding/parsing Ethernet frame headers (Destination MAC, Source MAC, Type field)
    • Check and generate CRC (FCS)
  2. Data transmission and reception control
    • Read data from system memory (DMA), encapsulate it into Ethernet frames, and send it
    • Receive data from the PHY and write it into memory
  3. Traffic and buffer management
    • Send/Receive FIFO
    • Notify CPU via interrupt or polling
    • DMA descriptor management (driver layer focus)
  4. Link layer control
    • Support full-duplex / half-duplex
    • Support flow control (PAUSE frames)
    • Statistics (packet loss, collisions, error counts, etc.)
  5. Interface with PHY
    • Via MII / RMII / RGMII / SGMII etc. interfaces to communicate with PHY
    • Via MDIO/MDC Configure and manage PHY chip

PHY chip

The full name of the PHY chip is Physical Layer chip. The main task of the PHY chip is toConvert digital signals into analog signals suitable for transmission over physical media (such as network cables), thenconvert the received analog signal back to a digital signal, so that the computer can recognize and process it.

The core functions of PHY include:

  1. Signal encoding and decoding
    • 100M：MLT-3、4B/5B
    • 1000M：PAM-5
    • Adaptive equalization, echo cancellation
  2. Auto-Negotiation
    • Negotiation speed (10M / 100M / 1000M)
    • Negotiation duplex mode (half-duplex / full-duplex)
  3. Link detection
    • Network cable plug/unplug detection
    • Report Link Up / Link Down status to MAC
  4. Clock recovery and synchronization
    • Recover clock from the received signal
    • Ensure accurate data sampling
  5. MAC management interface
    • Using MDIO/MDC Register interface
    • Common in Linux driversphy_read()/phy_write()

Network transformer

Network transformers are also known as network isolation transformers, Ethernet transformers, etc. Network transformers mainly serve the functions of signal coupling, electrical isolation, and impedance matching. Located at Between the PHY and the RJ45 interface, it is a passive device.

The main functions of network transformers include:

  1. Electrical isolation
    • Providing isolation capability above 1500V
    • Preventing surges and potential differences from damaging the chip
    • Meeting Ethernet safety standards (such as IEEE 802.3)
  2. Signal coupling
    • Coupling the differential signal output by the PHY to the twisted pair
    • Block DC components, only transmit AC signals
  3. Impedance matching
    • Match PHY output impedance with network cable (100Ω)
    • Reduce reflection, improve signal quality
  4. EMI suppression
    • Reduce electromagnetic interference with common mode inductor
    • Enhance anti-interference capability

Inside many RJ45 interfaces Integrated network transformer and common mode inductor, commonly referred to in engineering as “RJ45 with LEDs”。

Connection method

Depending on the integration of CPU, MAC, and PHY, it can be further divided into the following three methods:

Method 1: CPU integrates MAC and PHY internally

In this method, the CPU integrates the MAC and PHY internally. Although this method can reduce the amount of peripheral hardware, the CPU cost will increase significantly, and because everything is integrated inside the CPU, the flexibility is poor. Therefore, this method is not commonly used in practical applications.

Method 2: MAC integrated inside the CPU, PHY uses an independent chip

The CPU only integrates the MAC controller internally, and the PHY uses an independent chip. The advantage of this method is that the PHY chip can be flexibly selected according to the application scenario, and it is alsothe current mainstream method。

The iTOP-RK3568 also uses this method. It should be noted that the RJ45 socket on the iTOP-RK3568 development board integrates a network transformer internally. If the used
RJ45 socket does not integrate a network transformer internally, a separate network transformer chip needs to be used.

Method 3: CPU does not integrate MAC and PHY, MAC and PHY use independent chips or integrated chips

Under this connection method, the CPU does not have MAC and PHY functions, which is what we often call the CPU itself not supporting network functions. If network functions are to be used, they need to be converted through other interfaces. Therefore, this method has a higher cost.

MDIO interface

The full name of the MDIO interface is Management Data Input/Output, which stands for Management Data Input/Output interface.

The main function isto realize the management and control of the PHY chip by the MAC controller, for example, configuring the operating rate by setting the PHY registers via the MDIO interface; it can also obtain the status of the network port by reading the status registers of the PHY chip.

The MDIO interface is asynchronous serial half-duplex interface, consisting of two signal lines, which are:

  • MDIO: Acts as a data line, used to transmit configuration and management data between the MAC layer and the PHY layer.
  • MDC: Acts as a clock line, providing clock synchronization for data transmission.

PHY registers

In the 802.3 specification, the first 16 registers of the PHY chip are defined, as shown in the figure below:

In development, we deal with Register 0 and Register 1 the most.

Register 0 is the control register: used to configure the operating mode and parameters of the PHY chip. The control register can be used to set the chip’s transmission rate (10Mbps, 100Mbps, or 1000Mbps), duplex mode (full-duplex or half-duplex), and enable or disable the auto-negotiation function, etc.

Register 1 is the status register: Used to indicate the current working status of the PHY chip. For example, through the status register, you can check whether the network connection is successful, whether a link fault is detected, the current transmission rate, and the duplex mode, etc.

Registers 16 to 31 are extended registers: Registers 0 to 15 are registers specified by IEEE 802.3. Regardless of which manufacturer’s PHY chip is used, registers 0 to 15 are exactly the same. Therefore, in general, using the generic PHY driver in Linux is sufficient to drive the PHY chip. However, PHY chips from different manufacturers may have additional features, so the extended registers are the registers left for PHY manufacturers to freely implement. In other words, extended registers are only involved when these specific features are used.

PHY address

Each PHY register has a unique address. Through these addresses, we can accurately access and operate the corresponding registers. In the iTOP-RK3568 development board, the address of the RTL8211 PHY chip can be set through pins 22, 27, and 26.

Physical address (MAC address)

A physical address, also known as a MAC address, is the unique identifier of a network device. It consists of 48 binary bits, usually represented by 12 hexadecimal digits, such as 00:11:22:33:44:55.

IP address

An IP address is the address of the network layer, used to identify the logical location of a device in the network. It is divided into two versions: IPv4 and IPv6:

  • An IPv4 address consists of 32 binary bits, usually represented in dotted decimal notation, such as 192.168.1.1;
  • An IPv6 address consists of 128 binary bits. The main function of an IP address is to enable communication between different networks. Through the IP address, data packets can be routed from the source network to the target network.

PHY address

A PHY address is the address used to uniquely identify a PHY chip in a network. In a system, there may be multiple PHY chips present at the same time. Through the PHY address, different PHY chips can be distinguished so that they can be individually configured and managed.

Fundamentals of Network Application Programming

Byte Order

Little-endian mode: the low-order byte is stored at the low address, and the high-order byte is stored at the high address.

Big-endian mode: the high-order byte is stored at the low address, and the low-order byte is stored at the high address.

Byte order conversion functions

There are four common byte order conversion functions: htonl, htons, ntohl, and ntohs. A brief introduction to each function is shown in the table below:

Their core function is to resolve the byte order incompatibility between different hosts and networks.

uint32_t htonl(uint32_t hostlong)

  • Function: Used to convert a locally stored 32-bit integer into a format suitable for network transmission (big-endian mode).
  • Parameter: 32-bit host byte order integer.
  • Return value: Returns a 32-bit network byte order integer.

uint16_t htons(uint16_t hostshort)

  • Function: Used to convert a locally stored 16-bit integer (such as a port number) into a format suitable for network transmission.
  • Parameter: 16-bit host byte order integer.
  • Return value: Returns a 16-bit network byte order integer.

uint32_t ntohl(uint32_t netlong)

  • Function: Used to convert 32-bit data received from the network (such as an IPv4 address) into a format that the local system can process.
  • Parameter: 32-bit network byte order integer.
  • Return value: Returns a 32-bit host byte order integer.

uint16_t ntohs(uint16_t netshort)

  • Function: Used to convert 16-bit data received from the network (such as a port number) into a format that the local system can process.
  • Parameter: 16-bit network byte order integer.
  • Return value: Returns a 16-bit host byte order integer.

UDP

The full English name of UDP is User Datagram Protocol, meaning User Datagram Protocol, it is aconnectionlesstransport layer protocol, which is part of the TCP/IP protocol stack. It provides a simple, lightweight data transmission method suitable for scenarios that require high speed but low reliability, such as video conferencing, online gaming, voice calls, etc. These scenarios are latency-sensitive and can tolerate occasional packet loss.

TCP

The full English name of TCP is Transmission Control Protocol, which means Transmission Control Protocol, it is aconnection-orientedtransport layer protocol, like UDP, both belong to the core part of the TCP/IP protocol stack.

TCP ensures the integrity, order, and non-loss of data during transmission by establishing reliable end-to-end connections. It uses a three-way handshake to establish connections and ensures the reliability of data transmission through mechanisms such as acknowledgment, retransmission, and flow control.

Therefore, TCP is suitable for scenarios that require high data accuracy, such as file transfer, email, and web browsing, where the integrity and order of data are crucial, and the tolerance for latency is relatively high.

  • Compared to a UDP client, a TCP client has an additional step in between: calling connect to initiate a connection request.
  • Compared to a UDP server, a TCP server has two additional steps in between: calling listen to start listening mode, and calling accept to block and wait for client requests.
    • listen sets the socket to passive mode to wait for client connection requests.
    • When a client initiates a connection, accept blocks and waits until it receives a request, then returns a new socket file descriptor specifically used for communicating with that client.

After the client initiates a connection request via connect, it performs a TCP three-way handshake with the server to establish the connection. Once the connection is established, the client and server can send and receive data using write() and read().

After data transmission is complete, the client and server close the connection via close() respectively.
