Problem:

P45 Jump GameII

https://leetcode.cn/problems/jump-game-ii/description/?envType=study-plan-v2&envId=top-interview-150

The first thing that comes to mind is dynamic programming
Let dp[i] represent the minimum number of jumps to reach i, with -1 indicating unreachable. Then for 0 <= j < i, if j can reach i, i.e., nums[j] + j >= i and dp[j] != -1

dp[i]=min(dp[j]+1) dp[i] = min (dp[j] +1) dp[i]=min(dp[j]+1)

If there is no reachable j, then

dp[i]=−1 dp[i] = -1 dp[i]=−1

This problem is actually a classic greedy problem. Looking from back to front, the last position must have been jumped from some previous position, but there could be multiple such previous positions. We greedily choose the farthest one (since it’s an extra step anyway, choosing the one closer to the start will definitely reduce the number of jumps, because if two points can reach the last position, a is further ahead and b is further back, we will definitely choose a, because the steps to reach a must be less than or equal to the steps to reach b). After choosing, this chosen position becomes the new last position.

It can also be modified using the idea of the following problem

P55 Jump Game

https://leetcode.cn/problems/jump-game/description/?envType=study-plan-v2&envId=top-interview-150

Official solution:

If we perform a forward search “greedily”, finding the farthest reachable position each time, we can obtain the minimum number of jumps in linear time.

For example, for the array [2,3,1,2,4,2,3], the initial position is index 0. Starting from index 0, the farthest reachable position is index 2. Among the positions reachable from index 0, the value at index 1 is 3, and starting from index 1 can reach a farther position, so the first step goes to index 1.

Starting from index 1, the farthest reachable position is index 4. Among the positions reachable from index 1, the value at index 4 is 4, and starting from index 4 can reach a farther position, so the second step goes to index 4.

In the specific implementation, we maintain the maximum index position that can currently be reached, denoted as the boundary. We traverse the array from left to right, and when we reach the boundary, we update the boundary and increment the jump count by 1.

When traversing the array, we do not visit the last element. This is because before visiting the last element, our boundary must be greater than or equal to the last position; otherwise, we would not be able to jump to the last position. If we visit the last element, in the case where the boundary is exactly the last position, we would add an “unnecessary jump count”, so we do not need to visit the last element.
