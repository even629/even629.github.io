Exercise 4: ldr pseudo-instruction

  • Instruction: Each instruction corresponds to a CPU operation
  • Pseudo-instruction: A command issued to the compiler, it is an operation processed by the assembler during the assembly of the source program. They can complete functions such as processor selection, defining program modes, defining data, allocating storage areas, indicating the end of the program, etc. In short, it can be decomposed into a set of several instructions.
  • The ldr instruction can be either a wide-range address read pseudo-instruction or a memory access instruction. When there is a ‘=’ before its second parameter, it represents a pseudo-instruction; otherwise, it represents a memory access instruction.
  • The ldr pseudo-instruction has no immediate access restrictions

Additional Notes

  • Unsigned Load (LDR, LDRB, LDRH): High-order bits filled with0。
  • Signed Load (LDRSB, LDRSH, LDRSW): Sign extension is performed (if the most significant bit is 1, it is extended as 1).
  • If accessing and storing 4 bytes or 8 bytes both use ldr and str, except that the target register uses wn or xn
  • STRThis series of instructions does not have a signed version, because when storing, they are written to memory “as is”,without performing sign extension, zero extension, or any padding operations, but note the register bit width, which is different for xn and wn.
  • AArch64 defaults to little-endian (low address stores low byte).
  • There is also a class ofatomic load/store instructions（LDAXR、STLXRetc.) used for lock operations

Exercise 3: Why does it run wild after ret

Traps and pitfalls

bl instruction: used to call subroutines, it writes the return address to the x30 register, the return address is PC+4

Calling bl in a function to call a subroutine may overwrite the parent function’s lr register, and then the parent function’s ret will run wild

Simply put, the subroutine changed the value of the x30 register and did not restore it upon return.

Memory exclusive load and store instructions

  • ldxr instruction

Memory exclusive load instruction, loads the memory address from memory in an exclusive manner into a general-purpose register

  • stxr instruction

Memory exclusive store instruction

  1. ldrx loads memory, but it monitors the access to this memory through an exclusive monitor. The monitor marks this memory address as exclusive access, ensuring it is accessed in an exclusive manner

  1. stxr is a conditional store to memory. The memory address marked by ldxr just now is stored in an exclusive manner. Note that the first register is w0

  • ldxr instruction and stxr instruction usually need to be used in pairs
  • Linux kernel is often used to implement atomic access, such as atomic_write()，atomic_set_bit()
  • The spinlock mechanism can be simply implemented using ldxr and stxr instructions

Overview Summary

  • The instruction set running in the aarch64 execution state, where 64 refers to its execution environment, not the instruction length

  • The instruction length is 32bit, not 64bit

  • 31 general-purpose registers, xn is 64bit, wn is 32bit

  • Zero register: xzr, wzr

  • pc is not a general-purpose register and usually cannot be returned directly

  • x30 is used as the link register lr, for function return

  • ELR_ELx is used for returning from exceptions

  • Each exception level has its own stack SP, e.g., SP_EL0, SP_EL1

  • SP is not a general-purpose register

  • Registers for SIMD and floating-point operations

  • Qn (128bit, 16 bytes), Dn (64bit), Sn (32bit), Hn (16bit), Bn (8bit)

  • PSTATE（Processor State Register）Not a single physically existing register, but ratherA set of processor states composed of multiple bit fields。

  • PSTATEThe value of controls the current state of the CPU, including exception masking, current exception level, condition codes, etc.

Summary of Aligned Access

  • For normal memory, unaligned access is supported. (MMU needs to be enabled, and the memory attribute set to normal)
  • It can be individually configured to trigger an exception on unaligned access (set SCTLR_Elx.A)
  • For Device memory, unaligned access will trigger a Data Abort exception
    • Systems without MMU enabled, for example, our experimental code accessing DDR programmed to access device memory
  • Instruction prefetch requires 4-byte alignment, otherwise an exception will be triggered
  1. Normal memory supports unaligned access
  • Condition：
    • Enabled MMU（Memory Management Unit）；
    • And the memory region’s attribute is set to Normal memory；
  • Result：
    • Supports unaligned access for words (4 bytes), halfwords (2 bytes), and bytes (1 byte);
    • Enables cross-boundary access without triggering exceptions;
  1. Configure unaligned access exception (SCTLR_ELx.A bit)
  • SCTLR_EL1.A Controls whether to allow unaligned access（Alignment Check）：
    • A = 0(Default): Allows unaligned access (if memory type permits);
    • A = 1：Forces aligned access, once unaligned access occurs, triggers Alignment fault exception (Data Abort)；
  1. Device memory does not support unaligned access
  • Characteristics：
    • Regardless of whether the MMU is enabled, Device memory typedoes not allow unaligned access；
  • Consequences：
    • Once an unaligned address is accessed (e.g., accessing0x100332-bit data), it will trigger Data Abort exception；
  • Application example：
    • Direct mapped access in experimental environment DDR, peripheral registers , etc., are usually defined as Device memory type;
    • Access to these areas must be aligned, e.g., accessing 32-bit data must be 4-byte aligned;
  1. Instruction prefetch requirement
  • Instruction fetch must be 4-byte aligned(i.e., the lowest two bits of the address must be 0);
  • otherwise it will trigger Instruction Abort exception；
  • Reason: ARMv8 instructions are stored and fetched in 4-byte units, and cannot be decoded and executed from unaligned positions.

Pitfall 3: Store instruction data size

When using the str instruction to set registers, be sure to pay attention to the bit width of the register, otherwise it will cause a crash.

The bit width of peripheral registers on the Raspberry Pi 4b is all 32-bit, which is 4 bytes.

Pitfall 4: The big pitfall of ldxr

There are many restrictions on the use of the ldxr instruction

  1. First, ensure that the accessed memory is normal memory and is shareable

  1. If accessing device memory, for example when the MMU is not enabled, the CPU IP core needs to support accessing device memory in an exclusive manner. You need to check the description in the specific CPU IP manual.

For example: Chapter 6.45 of the Cortex-A72 MPCore Processor Technical Reference Manual

Solution:

Fill in the page table, enable the MMU and cache, before you can use ldxr and stxr
