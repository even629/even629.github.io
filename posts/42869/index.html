This article introduces the compilation and running process of a minimal Linux system, detailing error handling and patch application when compiling Busybox in a specific environment, the creation and packaging of root filesystem configuration files, and the steps for compiling the Linux kernel. Additionally, it summarizes the configuration method for reading source code and device trees using Emacs with clangd and dts-lsp, and presents the final result of running the system via QEMU.

Compilation

make menuconfig Failed

During the make menuconfig step, if compiling under Arch Linux or using a high-version gcc, an error may occur:

Even if ncurses is already installed, an error will still be reported

Reference solution:

Busybox build fails with ncurses header not found in archlinux

https://stackoverflow.com/questions/78491346/busybox-build-fails-with-ncurses-header-not-found-in-archlinux-spoiler-i-alrea

The discussion above says that a higher version of GCC has a breaking change, i.e., the main function must have a return type

tc build error

If using a higher version of the Linux kernel, you need to disable tc, refer to

[Bug 15934] New: Busybox fails to build with linux kernels >= 6.8

https://lists.busybox.net/pipermail/busybox-cvs/2024-January/041752.html

Disable it in make menuconfig
