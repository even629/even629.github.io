1-Wire Introduction

1-Wire Overview

1-Wire is a serial communication protocol and hardware bus, used for transmitting data and control signals between electronic devices. It was developed by the independent chip manufacturer Dallas Semiconductor and has been widely used in various applications.

Unlike serial data communication methods such as SPI and I2C, the characteristic of 1-Wire isit only requires one signal line, which can transmit both clock and data, and the data is bidirectional. Therefore, 1-Wire has the characteristics of saving IO ports, simple structure, and easy expansion and maintenance.

The single bus is used in various applications, includingtemperature sensors, humidity sensors, EEPROM memory, clocksetc. It has been widely used in many fields, such as industrial automation, home automation, the Internet of Things, and electronic device monitoring. This article uses the DS18B20 temperature sensor as an example to learn about the single bus.

  1. Signal line: The single bus uses a single signal line for data transmission and communication. This line is called the data line and also serves as the power supply line. All devices on the single bus are connected to this signal line.
  2. Pull-up resistor: The single bus requires a pull-up resistor connected between the signal line and the power supply to ensure that when no device is sending data, the level on the signal line remains high (logic 1). The value of the pull-up resistor is typically between 4.7 kilohms and 10 kilohms. (GPIO set to pull-up)
  3. Device: The single bus supports multiple devices connected to the signal line. Each device has a unique 64-bit address, which is used to identify and select the target device for communication. Devices can be various types of sensors, memory, clocks, etc.
  4. Processor: The processor is the controller on the single bus, responsible for sending commands, reading responses, and controlling the slave devices on the single bus.

Communication steps of the single bus

The single bus is amaster-slave structure，The slave will only respond when the master calls it., sothe master must strictly follow the command timing of the single bus. If the command timing is incorrect, the device will not respond.

The communication steps of the single bus usually include the following stages.

  1. Initialization: Before communication starts,the master device sends an initialization signal to ensure that no other devices are communicating on the single bus. The initialization signal is a specific sequence, usually pulling the data line low for a period of time and then releasing it.
  2. ROM operation command
  3. Function command

DS18B20 Introduction

DS18B20

Chip Overview

The DS18B20 is a digital temperature sensor chip,Provides temperature measurement with 9 to 12-bit resolution, which can be achieved throughprogrammable non-volatile memory unitsto implement lower and upper temperature limit alarms.

It is a device based on the single-bus communication protocol, requiring only one signal line and one ground line.

The DS18B20 can measure temperature with high precision up to 0.0625°C. It has a wide measurement range, typically between -55°C and +125°C.

The DS18B20 chip can be powered via the single bus from the host device, or by an external power supply. This allows for flexible power supply options in some low-power applications.

Each DS18B20 has a globally unique 64-bit serial number, allowing multiple DS18B20s to be networked on the same single bus in series. Only one processor is needed to control multiple DS18B20s distributed over a large area. This networking method is particularly suitable for applications such as HVAC environmental control, building, equipment, grain condition monitoring, industrial temperature measurement, and process monitoring and control.

Basic Performance

The DS18B20 is a digital temperature sensor chip with the following basic performance characteristics:

  • Uses a single-bus interface requiring only one port pin for communication
  • Each chip has a globally unique 64-bit serial number
  • Features multi-point distributed temperature sensing without the need for peripheral components
  • Can be powered via the data line, with a supply voltage range of 2.5V to 5.5V
  • Temperature measurement range is -55°C to +125°C
  • Accuracy of ±0.4°C in the range of -10°C to 70°C
  • Temperature resolution 9-12 bits selectable
  • At up to 12-bit accuracy, temperature conversion speed is less than 750ms
  • Has user-defined non-volatile temperature alarm settings
  • Alarm search command identifies and flags devices exceeding the programmed temperature limit
  • Strong electrostatic protection capability: HBM 8000V, MM 800V
  • Available in MSOP8, SOP8 surface-mount packages and 3-pin TO-92, TO-92S packages.

Pin configuration and package

DS18B20 chip pin configuration:

  • VDD: Power supply pin, used to provide the positive power supply for the chip.
  • DQ: Data pin, used for 1-Wire communication and data transmission.
  • GND: Ground pin, connected to the ground (negative power supply) of the chip.

The DS18B20 chip can be used in different package types, the most common of which are the TO-92 package and the TO-92-3 package. These packages are all small packages with three pins, suitable for through-hole mounting and surface mounting.

The TO-92 package is a common small plastic package, with pins arranged in order as VDD, DQ, and GND.

The TO-92-3 package is similar to the TO-92 package, but the pin order is slightly different. The pin order of the TO-92-3 package is GND, DQ, and VDD.

In addition to these common package types, the DS18B20 can also be used in other package types, such as SOT-23 and TO-263 packages, etc. These package types may have different pin configurations and dimensions. As shown in the figure below:

Internal Structure

The DS18B20 is a digital temperature sensor chip, and its internal structure mainly includes the following components, as shown in the figure below:

  • Temperature Sensor: The DS18B20 integrates a temperature sensor internally, which is used to measure the ambient temperature. The sensor typically detects temperature based on changes in a reference voltage and converts it into a digital signal.
  • A/D Converter: The DS18B20 chip contains an analog-to-digital converter (A/D converter) internally, which is used to convert the analog temperature value measured by the sensor into a corresponding digital representation. This allows temperature data to be processed and transmitted in digital form.
  • Memory: The DS18B20 chip also has internal memory, which is used to store configuration information and temperature measurement results. The memory can store a unique 64-bit address, temperature resolution, and other related settings.
  • Control Logic: The DS18B20 chip includes control logic circuitry, which is used to manage temperature measurement, communication, and other related functions. The control logic coordinates the operations of various parts and communicates with the host device.
  • Single-bus Interface: The DS18B20 adopts a single-wire communication protocol, and its internal structure includes a data line and a pull-up resistor for communicating with the host device. The single-bus interface simplifies the wiring for connection and communication, allowing multiple DS18B20 sensors to be easily daisy-chained on the same bus.

Register Introduction

  • Temperature Register: The temperature register stores the result of the most recent temperature measurement. It is a 16-bit register containing the raw data of the temperature value. By reading the data in the temperature register and combining it with the resolution setting, the actual temperature value can be calculated.
  • Configuration Register: The configuration register is used to set the operating mode and temperature resolution of the DS18B20. It is an 8-bit register, with each bit corresponding to a configuration option. By writing to the configuration register, you can select the temperature resolution, trigger temperature conversion, and use the power supply mode, etc.

Temperature register

The temperature register of the DS18B20 chip is a 16-bit register used to store the raw data of the most recent temperature measurement. The bit layout of the temperature register is shown below:

The least significant bit (LSB) of the temperature register is the 2−4 2^{-4} 2−4 bit, indicating a minimum temperature accuracy of 0.0625°C. The other bits sequentially represent higher temperature accuracies, which are:

  • 2−3 2^{-3} 2−3（0.125）
  • 2−2 2^{-2} 2−2（0.25）
  • 2−1 2^{-1} 2−1（0.5）
  • 20 2^{0} 20（1）
  • 21 2^{1} 21（2）
  • 22 2^{2} 22（4）
  • 23 2^{3} 23（8）

The raw data stored in the temperature register of the DS18B20 chip can be used to calculate the actual temperature value through the following steps:

  1. The 16-bit data read from the temperature register can be interpreted as a signed integer, where the most significant bit (MSB) represents the sign bit. If the sign bit is 0, it indicates a positive temperature; if the sign bit is 1, it indicates a negative temperature.
  2. Extract bits 4 to 15 from the temperature register, totaling 12 bits, which represent the integer value of the temperature (including sign information).
  3. Combine these 12 bits of data with the sign bit to form a signed integer.
  4. Based on the selected temperature resolution,multiply the signed integer by the corresponding resolution factorto obtain the actual temperature value.
  5. When the temperature is greater than 0, the sign bit is 0, and these 12 bits of data are directly interpreted as a signed integer (with the high bits padded with 0), then multiplied by the resolution factor to obtain the actual temperature.
  6. When the temperature is less than 0, the sign bit is 1, and these 12 bits of data are stored in two’s complement form. Take its absolute value (invert the lower 11 bits and add one), multiply by the resolution factor, and then negate it to obtain the actual temperature.

For example, assume a 12-bit temperature resolution is selected, and the data read from the temperature register is 0x1FFF.

The binary representation of 0x1FFF is: 0001 1111 1111 1111. The most significant bit is 0, indicating a positive temperature. Take the lower 11 bits: 111 1111 1111. Combine these 11 bits with the sign bit to get the signed integer: 0111 1111 1111 (corresponding to 0x07FF).

For a 12-bit temperature resolution, the resolution factor is 0.0625°C. Multiply the signed integer 0x07FF by the resolution factor: 0x07FF * 0.0625 = 127.9375°C.

Therefore, the data 0x1FFF read from the temperature register corresponds to an actual temperature value of approximately 127.94°C.

Instruction Introduction

According to the DS18B20 communication protocol, the host (MCU) must go through three steps to control the DS18B20 to complete temperature conversion:

  • A reset operation must be performed on the DS18B20 before every read and write.
  • After a successful reset, send a ROM command.
  • Finally, send a RAM command, so that the predetermined operation can be performed on the DS18B20.

The reset requires the master CPU to pull the data line low for 500 microseconds, then release it. After receiving the signal, the DS18B20 waits for about 16 to 60 microseconds, and then issues a presence low pulse of 60 to 240 microseconds. The master CPU receiving this signal indicates a successful reset.

ROM Command

After the host detects the presence pulse, it can send a ROM command. These commands act on the unique 64-bit ROM codeof each slave device, allowing the host to select a specific device when multiple devices are present on the 1-Wire bus.

These commands also allow the host to:

  • determine how many devices are on the bus
  • confirm the device type
  • detect if any device has triggered an alarm state

There are a total of 5 types of ROM commands, and the length of each command is 8 bits。

Before sending a function command to the DS18B20, the host must first send an appropriate ROM command.

Search ROM [F0h]

After the system is powered on, the host must identify the ROM codes of all slave devices on the bus, so that it can know:

  • How many devices there are
  • The type of each device

The host completes this process through a “process of elimination”, which requires repeated execution ofSearch ROM command + data exchangemultiple times until all devices are found.

If there is only one device on the bus, a simplerRead ROM [33h]can be used instead of Search ROM.

After each Search ROM process is completed, the host must return to step 1 (initialization) of the communication flow.

Read ROM [33h]

This command can only be used when there is only one slave device on the bus。

It allows the host to directly read the 64-bit ROM code of the device without executing the Search ROM process.

If there are multiple devices on the bus but this command is used:

  • All devices will respond simultaneously
  • Data collision will occur

Match ROM [55h]

Host sendsMatch ROM + 64-bit ROM codeto precisely select a specific device.

Only devices with a perfectly matching ROM code will respond to subsequent function commands, while all other devices will wait for the next reset.

Applicable to:

  • Multi-device bus
  • Single-device bus

Skip ROM [CCh]

This command will:

  • Select all devices on the bus simultaneously
  • No need to send ROM code

For exampleSkip ROM+Convert T [44h]can make all DS18B20s start temperature conversion simultaneously.

NoteSkip ROM+Read Scratchpad [BEh]Can only be used when there is only one device on the bus.

If there are multiple devices on the bus but this command is used:

  • All devices will respond simultaneously
  • A data collision will occur

Alarm Search [ECh]

This command operates in the same way as Search ROM,

but only devices that have triggered the alarm flag will respond. It is used to detect if any device exceeded the alarm threshold during the most recent temperature conversion

After each Alarm Search process is completed, the host must return to the initialization step.

Function Command

Convert T [44h]

This command initiates a temperature measurement.

After the conversion is complete:

  • The temperature result is written to the 2-byte temperature register in the scratchpad
  • The chip enters a low-power idle state

If it is parasitic power mode：

  • Within 10µs after the host sends the command,
  • turn on the strong pull-up
  • for the entire conversion time tCONV

If it is external power mode：

the host can continuously read time slots:

  • Returns 0 → still converting
  • Returns 1 → conversion complete

This query method cannot be used with parasitic power because the bus is pulled high by the strong pull-up.

Write Scratchpad [4Eh]

Allows the host to write 3 bytes to the scratchpad:

  • 1st byte → TH (byte 2)
  • 2nd byte → TL (byte 3)
  • 3rd byte → Configuration register (byte 4)

Data must:

  • LSB first
  • The three bytes must be written at once before resetting, otherwise the data may be corrupted.

Read Scratchpad [BEh]

Host reads scratchpad contents:

  • Starting from the least significant bit of byte 0
  • Read up to byte 8 (CRC)

If only partial data is needed:

  • Can reset at any time to terminate reading

Copy Scratchpad [48h]

Copy the scratchpad’s:

Copy to EEPROM.

In parasitic power mode:

  • Strong pull-up must be enabled within 10µs
  • Last for at least 10ms

Recall E2 [B8h]

Read from EEPROM:

Reload to scratchpad.

Host can query the status:

  • 0 → Recovering
  • 1 → Complete

This operation is executed automatically upon power-up, so the scratchpad has valid data as soon as it is powered on.

Read Power Supply [B4h]

Detect power supply mode:

Read a time slot after the host sends a command:

  • Bus pulled low → Parasitic power
  • Bus remains high → External power

Operation Example

Only one DS18B20 on the single bus

There is only one DS18B20 on the bus, and it uses parasitic power. The host writes TH, TL, and the configuration register to the scratchpad of this DS18B20, then reads the scratchpad and recalculates the CRC to verify the data. After that, the host copies the contents of the scratchpad to the EEPROM.

Communication timing

The DS18B20 uses a strict 1-Wire communication protocol to ensure data integrity. This protocol defines several signal types:

  • reset pulse
  • presence pulse
  • write 0
  • write 1
  • read 0
  • read 1

All data on the 1-Wire bus is LSB first (least significant bit first)

Reset timing

1-WireAll communication on the bus starts with an initialization sequence .

This initialization sequence includes:

  • a reset pulse sent by the host
  • followed by a presence pulse sent by the slave device

The function of the presence pulse is to tell the master: there is a slave device (e.g., DS18B20) on the bus, and they are ready to communicate.

During the initialization process:

  1. The master sends a reset pulse, pulling the 1-Wire bus low for at least 480µs
  2. The master releases the bus and enters receive mode; after release, the bus is pulled high by the 5kΩ pull-up resistor
  3. The DS18B20 detects the rising edge and waits 15µs ~ 60µs
  4. The DS18B20 sends a presence pulse, pulling the bus low 60µs ~ 240µs
  5. The bus is pulled high by the pull-up resistor (RESISTOR PULLUP)

Read/Write Timing

  • The master in write time slot writes data to the DS18B20,
  • in read time slot reads data from the DS18B20.

On the 1-Wire bus: each time slot transmits only 1 bit of data.

Write Time Slots

There are two types of write time slots:

  • Write 1
  • Write 0

The host uses: Write 1 time slot to write a logic 1, Write 0 time slot to write a logic 0

All write time slots must satisfy:

  • Each time slot ≥ 60µs and ≤ 120µs
  • Recovery time between adjacent time slots ≥ 1µs

Both types of write time slots are started by the host: first pulling the 1-Wire bus low

  • Rules for Write 1

    • After the host pulls the bus low: it must release within 15µs

    • After release: the 5kΩ pull-up resistor will pull the bus high

  • Rules for Write 0

    • After the host pulls the bus low: it must maintain the low level for ≥ 60µs

The DS18B20 will detect the bus state in a sampling window: after the host starts the write time slot 15µs ~ 60µs During this interval, if this time:

  • Bus is high → Write 1
  • Bus is low → Write 0

Read Time Slots

DS18B20 Data can only be sent during a read slot. Therefore:

  • If the master wants to read data, it must actively create a read slot
  • The slave cannot speak on its own

This is a:Master-driven synchronous communication

The master will read immediately after these commands:

  • Read Scratchpad [BEh]
  • Read Power Supply [B4h]

Additionally, the status can be queried using a read slot:

  • Convert T [44h] (whether temperature conversion is complete)
  • Recall E2 [B8h]

The read slot is initiated by the master:

  1. Master pulls the bus low ≥ 1µs
  2. Master releases the bus
  3. DS18B20 sends 0 or 1 in this window
    • Send 1 → do nothing → line is pulled up → high level
    • Send 0 → actively pull down the bus, (DS18B20 releases the bus at the end of the time slot, pull-up resistor restores high level)

The entire Read Slot must be ≥ 60µs long，Recovery between time slots ≥ 1µs

Therefore, the DS18B20 data is valid within 15µs after the start of the read time slot, so the host must: complete sampling within 15µs
