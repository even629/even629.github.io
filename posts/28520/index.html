This article introduces common selection commands and basic operation shortcuts for the Emacs editor, discusses the storage location of Emacs configuration files on Windows systems and strategies for finding the HOME directory, and finally summarizes the potential impact of setting the HOME environment variable on other programs and recommendations for independent configuration using the registry.

Common Selection Commands

CommandNameEffect
M - hmark-paragraphSelect paragraph
C - x + C - pmark-pageSelect entire page
C - x hmark-whole-bufferSelect entire editor window content
M - yyank-popUse after C-y to paste previous items from clipboard

Location of Emacs configuration file on Windows

On Windows, the .emacs file may be called _emacs for backward compatibility with DOS and FAT filesystems where filenames could not start with a dot. Some users prefer to continue using such a name, because Explorer cannot create a file with a name starting with a dot, even though the filesystem and most other programs can handle it. In Emacs 22 and later, the init file may also be called .emacs.d/init.el. Many of the other files that are created by lisp packages are now stored in the .emacs.d directory too, so this keeps all your Emacs related files in one place.

All the files mentioned above should go in your HOME directory. The HOME directory is determined by following the steps below:

  1. If the environment variableHOME is set, use the directory it indicates.
  2. If the registry entry HKCU\SOFTWARE\GNU\Emacs\HOME is set, use the directory it indicates.
  3. If the registry entry HKLM\SOFTWARE\GNU\Emacs\HOME is set, use the directory it indicates. Not recommended, as it results in users sharing the same HOME directory.
  4. If C:.emacs exists, then use C:/. This is for backward compatibility, as previous versions defaulted to C:/ if HOME was not set.
  5. Use the user’s AppData directory, usually a directory called Application Data under the user’s profile directory, the location of which varies according to Windows version and whether the computer is part of a domain.

That is, Emacs configuration files come in three formats: .emacs file, _emacs file, or init.el file (for version 22 or higher) in the .emacs.d directory. However, regardless of the format, they must be placed in the correct directory for Emacs to find them, and that correct directory is the HOME directory.

Friends familiar with Linux generally know the home directory, simply represented by ~, or by the full path /home/, where xxx represents the username. So on Linux, just place .emacs and .emacs.d in this directory. However, Windows does not have the concept of a home directory, so Emacs searches for configuration files in the following way:

  1. If the HOME environment variable is set, use its value as the home directory ~
  2. If the registry key HKCU\SOFTWARE\GNU\Emacs\HOME exists, use its value as the home directory ~
  3. If the registry key HKLM\SOFTWARE\GNU\Emacs\HOME exists, use its value as the home directory ~ (the difference from 2 is that 2 is only for the current user’s registry path, while 3 is for all users)
  4. If C:.emacs exists, use C:\ as the home directory ~
  5. If none of the above exist, use\Users\AppData\Roaming as the home directory ~ (for XP and earlier Windows users, look under the Documents and Settings directory)

From the above strategies, 1 is a better practice, so it has the highest priority and is the recommended approach. Therefore, create a HOME environment variable and set its value to where you want to place Emacs configuration files.

Previously, I set the HOME directory using the above method, but now I regret it because I found that not only Emacs uses the HOME environment variable, but at least the following programs or plugins also use it:

  1. VIM, VIM places the _viminfo file in this directory;
  2. VIM’s neocomplcache plugin, it places a .neocon directory in it;
  3. Firefox’s pentadactyl plugin, it places a pentadactyl directory in it.

Therefore, personal suggestion:

  1. If you want a common directory to store these configurations, like the ~ directory under Linux, then setting the HOME environment variable is most suitable;
  2. If you want this directory to only store Emacs configurations, then it’s better not to use the HOME environment variable; instead, add a HOME key value in the registry. (At least I personally prefer this method)

That is:

at

HKEY_LOCAL_Create a new GNU entry under MACHINE\SOFTWARE, then create an Emacs entry under GNU, and then create a string value in Emacs with the key HOME and the value being the path to store.

Parameter control

Most Emacs commands accept numeric arguments, and for most commands, the function of these numeric arguments isto specify the repetition count of the command

First type C-u, then enter a number as the argument, and finally type the command

For example: C-u 8 C-f moves forward 8 characters

Some Emacs commands use numeric arguments for other purposes

For example: C-v and M-v

When given an argument, they scroll by the specified number of lines instead of screens

That is, C-u 8 C-v moves the cursor down 8 lines

M-k

Note: The difference between “kill” and “delete” is that killed text can be reinserted (at any position), while deleted text cannot be reinserted using the same method (though it can be undone via an undo command, as mentioned later). [In fact, although killed text appears to disappear, it is actually recorded by Emacs and can be retrieved; deleted text, while possibly still in memory, has been discarded by Emacs and thus cannot be retrieved.] Reinserting killed text is called “yanking.” Generally, commands that remove a lot of text record the removed text (they are set to be “yankable”), while commands that remove only a single character or whitespace do not record the removed content (so naturally you cannot yank it back).

Note: A single C-k removes the contents of a line, while a second C-k removes the newline character and moves all subsequent lines up. C-k handles numeric arguments in a special way: it removes the specified number of lines along with their trailing newlines, rather than simply repeating C-k. For example, C-u 2 C-k removes two lines and their newlines; this is clearly not the same as typing C-k twice.

Yank (paste)

The action of reinserting removed text is called “yanking.” (It’s like forcefully pulling back something that was taken away from you.) You can yank at the place where you removed the text, or elsewhere, and you can yank the same text multiple times to get multiple copies. Many other editors refer to killing and yanking as “cutting” and “pasting.”

Search

Emacs can search forward or backward for a string (a “string” refers to a sequence of consecutive characters).

The search command is acommand to move the cursor: after a successful search, the cursor stops at the location where the search target appears

Forward search

C-s

In an incremental search, Emacs tries to jump to the location where the search target appears. To jump to the next match, press C-s again. If the target is not found, Emacs beeps to inform you of the search failure. Throughout the process, you can use C-g to abort the search. [You will find that C-g returns the cursor to the starting position of the search, whileleaves the cursor at the search result, which is a very useful feature]

Backward search

File

Find a file

C-x C-f

Save file

C-x C-s

This command stores the text in Emacs into a file. The first time you save, Emacs renames the original file as a backup. The renaming rule is usually to add a ‘~’ character after the original filename. This feature can be turned off.

Buffer

Emacs places each file being edited in a place called a ‘buffer’. Every time a file is found, Emacs creates a buffer for it. Use the following command to list all current buffers:

C-x C-b List buffers

Type C-x 1 to leave the buffer list

No matter how many buffers exist, there can only be one ‘current’ buffer at any time, which is the one you are editing. If you want to edit another buffer, you must ‘switch’ to it. As mentioned earlier, using C-x C-f is one way. But there is a simpler way: using C-x b. With this command, you must enter the name of the buffer.

In most cases, a buffer has the same name as its corresponding file (excluding the directory name), but this is not absolute. The buffer list obtained with C-x C-b always shows the buffer name.

A buffer does not necessarily have a corresponding file. The buffer that displays the buffer list (called “Buffer List”) is like that. This TUTORIAL.cn buffer initially had no corresponding file, but now it does, because in the previous section you typed C-x C-s to save it as a file.

“Messages" buffer also has no corresponding file; this buffer stores messages that appear at the bottom of Emacs.

If you make some changes to a file and then switch to another file, this action does not save the previous file for you. The modifications to the first file still exist only in Emacs, that is, in its corresponding buffer. Moreover, changes to the second file will not affect the first file. This is useful, but it also means you need a convenient way to save the buffer of the first file. Switching back to that buffer and then using C-x C-s to save is too cumbersome. You need a simpler method, and Emacs has already prepared one for you:

C-x s Save multiple buffers

C-x s will find all buffers that have been modified but not yet saved, and then ask you one by one: Do you need to save?

Auto Save

If you have modified a file but your computer crashes before you save it, your changes are likely to be lost. To prevent such misfortune, Emacs periodically writes the file you are editing to an “auto-save” file. The auto-save file’s name has a “#” character at both the beginning and end. For example, if the file you are editing is called “hello.c”, its auto-save file is called “#hello.c#”. This file is deleted by Emacs after a normal save.

So, if the misfortune does happen, you can calmly open the original file (note: not the auto-save file) and then type M-x recover fileto recover your auto-save file. When prompted for confirmation, type yes。

Status Bar

The line directly above the echo area is called the “status bar”. The status bar displays some information, such as:

-😗*- TUTORIAL.cn 63% L749 (Fundamental)

  • The status bar shows the status of Emacs and some information about the text you are editing.
  • You should know what the filename means, right? It’s the file you found.
  • -NN%-- shows the position of the cursor in the entire text. If it is at the beginning of the file, it displays --Top-- instead of --00%–; if at the end of the file, it displays --Bot–. If the file is small enough that one screen can display all content, the status bar shows --All–.
  • “L” and the number following it give the line number of the cursor’s current line.
  • The asterisk (*) at the very beginning indicates that you have made changes to the text. A newly opened file has definitely not been modified, so the status bar shows a dash (-) instead of an asterisk.
  • The content in parentheses on the status bar tells you the editing mode currently in use. The default mode is Fundamental, which is what you are using now. It is a “major mode.”

Mode

Major modes are extension commands that can be started with M-x, and M-x fundamental-mode switches to Fundamental mode.

M-x text-mode . Switch to text mode

Use C-h m to view the documentation for the current major mode, and enter C-x 1 to close the documentation pane.

  • Major modes are called “major” because there are also “minor modes” that exist. Minor modes do not replace major modes but provide auxiliary functions. Each minor mode can be turned on or off independently, regardless of other minor modes or the major mode. So you can use no minor modes, or just one, or multiple minor modes simultaneously.
  • There is a minor mode called Auto Fill that is very useful, especially when editing natural language text. When Auto Fill is enabled, Emacs will automatically wrap lines for you when you type beyond the line boundary.
  • Use M-x auto-fill-modeto start Auto Fill mode. Using this command again will turn off Auto Fill mode. That is, if Auto Fill mode is not enabled, this command enables it; if it is already enabled, this command disables it. So we say this command can be used to toggle the mode.
  • The line boundary is usually set to 70 characters (referring to English characters here). You can reset it using the C-x f command with a numeric argument.
  • If you make some modifications in the middle of a paragraph, auto-fill mode will not re-wrap the entire paragraph for you; you need to manually re-wrap it with M-q. Note that the cursor must be within the paragraph you want to re-wrap.

Get more help

Emacs provides commands to view its command documentation, all of which start with CONTROL-h, which is therefore called the Help character.

The most basic help feature is C-h c. Type C-h c followed by a key sequence, and Emacs will give a brief description of that command.

Type C-h c C-p.

The message displayed should be like this:

​ C-p runs the command previous-line

This message shows the function name corresponding to the C-p command. Commands are implemented by functions, so the function name itself can be considered the simplest documentation—at least for commands you have already learned, their function names are sufficient to explain their functionality.

For more information, try replacing C-h c with C-h k.

Type C-h k C-p.

The above command will open a new Emacs pane to display the function name and its documentation. After reading, you can close the help pane with C-x 1. Of course, you don’t need to do this immediately; you can do something else in the editing pane first and then close the help pane.

C-h f explains a function. You need to enter the function name.

Type C-h f previous-line. Emacs will provide all the information it knows about the function that implements the C-p command.

C-h v is used to display documentation for Emacs variables. Emacs variables can be used to customize Emacs behavior. Similarly, you need to enter the variable name.

C-h a is Command Apropos. Enter a keyword, and Emacs will list all commands whose names contain that keyword. All these commands can be invoked with M-x. For some commands, Command Apropos will also list one or two key sequences.

Type C-h a file。

Emacs will display a list of M-x commands in another pane, containing all commands with “file” in their names. You can see key sequences like “C-x C-f” displayed next to command names like “find-file”.

C-h i reads the manual (commonly known as Info). This command opens a special buffer called “info” special buffer, where you can read the manuals for software packages installed on your system. To read the Emacs manual, press m emacsand that’s it. If you have never used the Info system before, press “?”, and Emacs will take you into the Info tutorial. After finishing this quick guide, Emacs Info will become your primary reference document.
