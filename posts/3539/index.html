Problem

Given two integersMandK, and an integer arraynums。

An integer sequenceseqIf the following conditions are met, it is called magic sequence:

  • seq's sequence length isM。
  • 0 <= seq[i] < nums.length
  • 2^seq[0] + 2^seq[1] + ... + 2^seq[M - 1]of binary representation hasKa set bit。

of this sequence array product is defined asprod(seq) = (nums[seq[0]] * nums[seq[1]] * ... * nums[seq[M - 1]])。

return all valid magic sequence’s array product of sum 。

Since the answer may be large, return the result modulo10^9 + 7modulo。

set bit refers to a bit with a value of 1 in the binary representation of a number.

What we need to do is:

  • Given an arraynums, lengthn
  • take a sequence of lengthMsequenceseq(elements from nums can be chosen repeatedly)
  • Condition:2^seq[0] + ... + 2^seq[M-1]binary hasKones
  • Array product:prod(seq) = nums[seq[0]] * ... * nums[seq[M-1]]
  • Goal: Sum of array products of all magic sequences (mod 1e9+7)

Number of sequence permutations

Suppose we take fromnumsindex0..n-1numbers:

  • Each numberitakesr_ia
  • Total satisfies:r_0 + r_1 + ... + r_{n-1} = M

Number of permutations of length M:

M!r0!r1!...rn−1!\frac{M!}{r_0! \, r_1! \, ... \, r_{n-1}!} r0​!r1​!...rn−1​!M!​

Formula for the number of permutations of length M

Number of permutations without considering duplicate numbers

  • If all numbers are distinct, the number of sequence permutations of length M is M!
    • Because M distinct elements can have M! permutations

Considering duplicate numbers

  • If some numbers appear repeatedly, for example:

seq=[a,a,b,b,b,c]seq = [a, a, b, b, b, c] seq=[a,a,b,b,b,c]

  • Swapping duplicate numbers does not produce a new sequence
  • In combinatorics, we need to - divide by the factorial of each duplicate number, to eliminate the effect of duplicate permutations
  • Suppose number i appearsr_itimes, the formula for the total number of permutations is:

排列数=M!r0!r1!…rn−1!\text{排列数} = \frac{M!}{r_0! \, r_1! \, \dots \, r_{n-1}!} 排列数=r0​!r1​!…rn−1​!M!​

This is the classicmultiset permutation formula: number of permutations of length M = factorial of total elements / factorial of each repeated element

array product formula

The array product of a sequence is defined as:

prod(seq)=nums[seq[0]]⋅nums[seq[1]]⋅⋯⋅nums[seq[M−1]]\text{prod(seq)} = nums[seq[0]] \cdot nums[seq[1]] \cdot \dots \cdot nums[seq[M-1]] prod(seq)=nums[seq[0]]⋅nums[seq[1]]⋅⋯⋅nums[seq[M−1]]

  • If the number i appears in the sequencer_itimes, then the product can be written as:

∏i=0n−1nums[i]ri\prod_{i=0}^{n-1} nums[i]^{r_i} i=0∏n−1​nums[i]ri​

  • Combining the number of permutations, the total contribution of a sequence { r0, r_1, r_2, … ,r } sequence is:

总贡献=M!r0!r1!...rn−1!⋅∏i=0n−1nums[i]ri\text{总贡献} = \frac{M!}{r_0! \, r_1! \, ... \, r_{n-1}!} \cdot \prod_{i=0}^{n-1} nums[i]^{r_i} 总贡献=r0​!r1​!...rn−1​!M!​⋅i=0∏n−1​nums[i]ri​

  • Explanation:
    • The array product corresponding to each sequence of length M is∏ nums[i]^r_i

    • Therefore multiplying them together is the total contribution of this combination

Dynamic programming approach

Use DP to simulate the carry rule of binary addition

  • Enumerate all possibler_i(number of occurrences of each number) is very complex
  • Dynamic programming approach:Consider the numbers in nums one by one

State definition

  • f[i][j][p][q]represents:
    • Already considered the firstinumbers (nums[0…i])
    • Total takenj
    • The sum of the lower i bits of the current mask (carry state) isp
    • The number of set bits in the lower i bits isq
    • Value = corresponding to all sequences∏ r_t! * nums[t]^r_t's sum, which is the sum of the contributions of all these cases

Transition formula

When considering thei+1number:

  • Assume selectedranums[i+1]
  • New state:

f[i+1][j+r][⌊p/2⌋+r][q+(pmod2)]+=f[i][j][p][q]∗nums[i+1]r∗r!f[i+1][j+r][\lfloor p/2 \rfloor + r][q + (p \bmod 2)] += f[i][j][p][q] * nums[i+1]^r * r! f[i+1][j+r][⌊p/2⌋+r][q+(pmod2)]+=f[i][j][p][q]∗nums[i+1]r∗r!

  • p/2andp%2The role of:
    • Simulate splitting the binary of mask from low to high bits
    • p % 2= number of set bits in the lower bits
    • p / 2= mask shifted left by one bit

Initialization

  • Fori=0, which is the first number:

f[0][j][j][0]=nums[0]j∗j!f[0][j][j][0] = nums[0]^j * j! f[0][j][j][0]=nums[0]j∗j!

  • Number of set bits in the lower 0 bits = 0

Result

  • Total set bits of mask = set bits of high mask + low q
