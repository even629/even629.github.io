This paper introduces the atomic operation mechanism under the ARM architecture, discussing in detail the working principle of Load-exclusive/Store-exclusive (LL/SC) instructions based on exclusive monitors in ARMv8, multi-core contention analysis, and low-power lock applications combined with the WFE instruction. Meanwhile, this paper summarizes the LSE (Large System Extensions) atomic instructions introduced in ARMv8.1, such as the implementation principles of atomic memory access operations like Compare and Swap.

Basic atomic operation functions in the Linux kernel

  • The Linux kernel provides atomic variables of type atomic_t, whose implementation depends on different architectures
  • Atomic operation functions of type atomic_t can guarantee the atomicity and integrity of an operation
  • “Read-Modify-Write” mechanism
    • Reading the value of an atomic variable into a general-purpose register
    • Modifying the value of the atomic variable in the general-purpose register
    • Writing the new value back to memory

If the CPU merely reads (loads) a variable’s value from memory, or merely writes (stores) a variable’s value to memory, it is uninterruptible

ARMv8 support for atomic operations

  • ARMv8 provides two ways of atomic operations
    • Traditional Load-exclusive and Store-exclusive methods
      • Supported on all ARMv8
      • LL/SC(Load-link/store-conditional)
    • LSE (Large System Extensions) supports atomic operation instructions
      • Supported starting from ARMv8.1, ARMv8.1-LSE
      • Added Compare and Swap instructions
      • Added Atomic memory operation instructions
      • Added Swap instruction

Load-exclusive and store-exclusive instructions

  • ldxrInstruction: Memory exclusive load instruction. Loads the value of a memory address into a general-purpose register in an exclusive manner from memory.
  • stxrInstruction: Memory exclusive store instruction. Stores new data into memory in an exclusive manner.
  • Load/Store Exclusive Pair instruction
  • Load/Store Exclusive instructions with acquire and release semantics

Example

  • Monitors the access to this memory through an exclusive monitor. The exclusive monitor marks this memory address as an exclusive access mode, ensuring that the memory address is accessed in an exclusive manner, unaffected by other factors.

Exclusive Monitor

  • The exclusive monitor has two states:
    • Open Access state
    • Exclusive Access state
  • When the ldxr instruction loads data from memory, the CPU marks this memory address as Exclusive Access state
  • When the CPU executes the stxr instruction, it needs to make a decision based on the state of the exclusive monitor
    • If the state of the exclusive monitor is Exclusive Access state, then the stxr instruction stores successfully, the stxr instruction returns 0, and the state of the exclusive monitor changes to Open Access state
    • If the state of the exclusive monitor is Open Access state, then the stxr store fails, and stxr returns 1

Notes

  • The exclusive monitor itself is not used to prevent CPU cores from accessing the marked memory, and does not lock the bus
  • The exclusive monitor merely serves a monitoring role, monitoring changes in state
  • The exclusive monitor cannot be viewed as a hardware lock

Architecture of the Exclusive Monitor

  • Usually, a system consists of multi-level exclusive monitors (defined during chip design)
    • Local exclusive monitor (Local monitor),Applicable to non-shareable memory
    • Cache-coherent global exclusive monitor (Internal coherent global monitor),Applicable to normal memory
    • External global exclusive monitor (External global monitor),Applicable to device memory
  • Some SoCs do not support the external global exclusive monitor. For example, the BMC2711 used on Raspberry Pi 4B
  • When the MMU is not enabled, accessing physical memory becomes accessing device memory. In this case, using the ldxr and stxr instructions will produce unpredictable errors.

  • There are many restrictions on the use of the ldxr instruction, requiring the memory to be normal memory and shareable.
  • If accessing device memory, such as when the MMU is not enabled, the CPU IP core needs to support accessing device memory in an exclusive manner. This requires consulting the description in the specific CPU IP manual.

Granularity of the exclusive monitor (Granularity of Exclusive Monitor)

  • The ERG (Exclusives Reservation Granule) in the CTR_EL0 register defines the minimum unit of the exclusive monitor.
  • The range that ERG can define is 4 words to 512 words, but it is usually the size of a cache line.
  • Example:
    • Assume the ERG is 2^4, which is 16 bytes. When using the ldxrb instruction to perform an exclusive read operation on address 0x341B4, the range from 0x341b0 to 0x341bf will be marked as exclusive access

WFE Wakeup

  • The CPU sleeping via WFE is woken up in the following ways:
    • unmasked interrupt
    • Event (Wakeup event)
  • Ways to trigger a wakeup event:
    • Executed the SEV instruction
    • The local CPU executed the SEVL instruction
    • Clear the exclusive monitor, transitioning from exclusive state to open state
  • When the CPU holding the lock writes to the lock region via the STLR instruction to release the lock, a wakeup event is triggered, and the CPU sleeping and waiting for the spinlock will be woken up

Atomic Memory Access

  • ARMv8.1 supports the following three atomic memory access operations (Large System Extensions)
    • Compare and Swap instructions, CAS and CASP
    • Atomic memory operation instructions
    • Swap instruction
  • Via ID_AA64ISAR0_The atomic field in the EL1 register is used to determine whether LSE is supported

Compare and Swap (CAS) instruction

  • Compare and Swap instruction: Checks if the value pointed to by ptr is equal to expected. If they are equal, assigns the value of new to ptr; otherwise, does nothing. Regardless of whether they are equal, it ultimately returns the old value of ptr

  • On ARMv8.1CAS instruction

If the value of Xn (Xn is an address) == Xs, then store the value of Xt in Xn, the return value is Xs, which equals the old value of Xn

cmpxchg atomically compares whether the value of ptr is equal to the value of old, if equal, sets the value of new to the ptr address, returns the value of old

  • Parameters:

    • x30: stores the old value, used for comparison

  • Function:

    1. Compare the value in memory withx30（old）
