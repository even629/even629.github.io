ADC Basics

Analog Signals and Digital Signals

Analog SignalIt is a continuously varying signal whose value can take any value within a certain range. It is continuous in both time and amplitude, and can accurately reflect changes in physical quantities in nature. For example, voltage signals output by temperature sensors, sound signals picked up by microphones, and light intensity signals detected by photoresistors all fall into the category of analog signals.

Digital SignalIt is a discrete signal whose value can only take specific values within a limited range, usually represented in binary form. Its characteristics are that it is discrete in time (there is a fixed time interval between sampling points) and discrete in amplitude (the value after quantization). For example, data stored and processed in computers, audio data in MP3 files, and pixel values in JPEG image files are all specific manifestations of digital signals.

DimensionAnalog Signal CharacteristicsCharacteristics of Digital Signals
EssenceContinuous variation, arbitrary valuesDiscrete values, often represented in binary
CharacteristicsTime + amplitude both continuousTime (sampling interval) + amplitude (quantized value) both discrete
AdvantagesAccurately reflect changes in natural physical quantitiesEasy to store / transmit / process, strong anti-interference capability
Typical scenariosSensor raw signals, natural sound/light/electric signalsComputer data, digital audio/video files

ADC Concept

ADC is Analog-to-Digital Converter is the abbreviation forAnalog-to-Digital Converter. It is an electronic device or circuit module used toconvert continuously varying analog signals (such as voltage, current, etc.) into discrete digital signals. These digital signals can be processed and stored by computers, microcontrollers, or other digital systems.

The process of an ADC converting analog signals to digital signals is typically divided intoSampling、Holding、QuantizationandEncodingfour steps. The definition and function of each step are shown in the table below:

StepDefinitionFunctionKey parameters or characteristics
SamplingThe process of measuring an analog signal at fixed time intervals.Discretizes the time-continuous analog signal, laying the foundation for subsequent digital processing.Sampling Rate, which is the number of samples per second. According to the Nyquist sampling theorem, the sampling rate must be at least twice the highest frequency of the signal.
HoldingAfter sampling is completed, the instantaneous value of the signal is held for a period of time for subsequent processing.Ensures that the signal does not distort due to rapid changes during the quantization process, improving the accuracy and stability of the conversion.No explicit parameters; the main purpose is to stabilize the signal.
QuantizationThe process of mapping the continuous amplitude obtained from sampling to a set of discrete values.Discretizes the amplitude-continuous signal so that it can be represented by a digital system.Resolution, usually expressed in bits (e.g., 8-bit, 10-bit, 12-bit, etc.). The higher the resolution, the more accurate the signal.
EncodingThe process of converting the quantized values into binary code.Represent discrete values in a form recognizable by digital systems, facilitating storage, transmission, and processing, enabling analog signals to be used by computers or other digital devices.No explicit parameters; the focus is on generating binary code suitable for use by digital systems.

ADCs are widely used in sensor signal acquisition, audio processing, image processing, industrial automation, and medical equipment:

  1. Sensor signal acquisition: Many sensors output analog signals. For example, environmental data collected by temperature sensors, pressure sensors, and photosensitive sensors usually exists in the form of analog signals.
  2. Audio processing: The sound signal picked up by a microphone is an analog signal.
  3. Image processing: The light signal captured by a camera is essentially an analog signal.
  4. Industrial automation: The measurement of various physical quantities (such as speed, position, pressure, etc.) usually relies on analog signals output by sensors.
  5. Medical equipment: Many biological signals (such as electrocardiograms, electroencephalograms, etc.) are collected in the form of analog signals.

ADC Resolution

The resolution of an ADC refers to the ability of the analog-to-digital converter to distinguish the smallest change in a signal. In other words, it is the fineness with which the ADC quantizes an analog signal into discrete digital values. In an ADC, a higher resolution means it can represent the input analog signal more accurately. When we restore a digital signal to an analog signal through a DAC (digital-to-analog converter), a high-resolution ADC can generate a result closer to the original analog signal.

The resolution of an ADC is usually expressed in bits, such as 8-bit, 10-bit, 12-bit, etc. The number of bits determines the number of discrete values the ADC can output; an n-bit ADC can represent 2^n different values. For example, an 8-bit ADC can represent 2^8=256 different values; a 10-bit ADC can represent 2^10=1024 different values; a 12-bit ADC can represent 2^12=4096 different values. The higher the number of bits, the higher the resolution, and the smaller the signal changes that can be distinguished.

Resolution can be calculated using the following formula:

分辨率=量程2n\text{分辨率} = \dfrac{\text{量程}}{2^n} 分辨率=2n量程​

Where:

  1. Range refers to the maximum range of the ADC input signal (i.e., the reference voltage range);
  2. n is the number of bits of the ADC;
  3. 2^n represents the total number of discrete values that the ADC can distinguish.

Example: Assume a 3-bit ADC has a reference voltage (range) of 2V, calculate its resolution:

  1. Range = 2V
  2. Number of bits is 3, 2^n = 2^3 = 8;

Substituting into the resolution calculation formula gives 2V/8 = 0.25V. This means the ADC can divide the input signal into 8 discrete levels, with a step size of 0.25V between each level.

In addition to resolution, other parameters of the ADC are as follows

ParameterDefinitionFunction or Significance
Sampling TimeThe time required to complete one sampling.Determines how fast the ADC can capture changes in the input signal. If the sampling time is too long, it may cause signal distortion or loss of detail.
Conversion TimeThe time required to convert an analog signal into a digital signal.Directly affects the speed and efficiency of the ADC. A shorter conversion time means the ADC can process signals faster.
Sampling FrequencyThe number of samples collected per second, commonly measured in ksps (kilo-samples per second) or Msps (mega-samples per second).Determines the ADC’s ability to capture signal changes. According to the Nyquist sampling theorem, the sampling frequency must be at least twice the highest frequency of the signal to accurately reconstruct the signal.
AccuracyThe error between the digital value output by the ADC and the actual analog input value.Reflects the reliability and measurement quality of the ADC, and is an important indicator for measuring its performance.

In addition to the SAR ADC, there are two TS-ADCs (Temperature Sensor ADCs), which are used to detect the temperature of the CPU and GPU respectively. If an excessively high temperature is detected for a period of time, an interrupt signal will be sent to the processor to take cooling measures; if the temperature remains too high, the entire chip will be reset, achieving a reboot effect. The TS-ADC is only responsible for detecting the temperature of the CPU and GPU, so it is not routed out on the baseboard.

Successive Approximation ADC

Successive Approximation ADC (Successive Approximation Register ADC, abbreviated as SAR ADC) is a common type of analog-to-digital converter, whose working principle is based on the successive approximation algorithm. It gradually compares the input analog signal with an internally generated reference voltage, ultimately determining a digital value close to the input signal. The schematic diagram of the SAR ADC working principle is shown below

From the above figure, it can be seen that the Successive Approximation Register ADC (SAR ADC) is mainly composed of the following parts:

  1. Comparator: Used to compare the input analog signal U1 with the reference voltage U0 output by the D/A converter (DAC), and generate a feedback signal.
  2. N-bit D/A Converter (DAC): Converts the digital value into an analog voltage, which is compared with the input signal to gradually approach the target value.
  3. N-bit Successive Approximation Register (SAR): Used to store the current comparison result, adjust the value based on the feedback from the comparator, and finally determine the digital output.
  4. Data Selection Logic: Controls the bit selection process of the successive approximation register, adjusting the output voltage of the DAC according to the binary search method.

Working Process:

  1. First, the analog input signal U1 enters the ADC and is temporarily stored by the sampling circuit for subsequent processing.
  2. Subsequently, the Successive Approximation Register (SAR) is initialized, and under the control of the data selection logic circuit, it gradually determines the final digital output value according to the binary search method.

During the conversion process, the SAR starts from the most significant bit (MSB) assuming it to be 1, and passes the current N-bit digital value to the D/A converter. The DAC converts it into an analog voltage U0, which is then sent to the comparator to be compared with the input signal U1. If U0 is less than U1, the current bit remains 1; otherwise, the current bit is changed to 0. This process is repeated, determining one bit at a time, until all N bits are completed, forming the final digital output.

  1. Finally, the SAR outputs the determined N-bit digital value, completing an analog-to-digital conversion.

An example is as follows:

The case of a 5-bit successive approximation ADC with a reference voltage of 32mV and a measured voltage of 21.5mV

  1. Setting the MSB: The SAR register sets the most significant bit (bit 5) to ‘1’ and the remaining bits to ‘0’. The current SAR value is 10000 (binary), which is 16 (decimal). The DAC generates a reference voltage U0=16×1mV=16 mV based on the SAR value. The comparator compares U1=21.5 mV and U0=16 mV. Since U1>U0, the current bit is kept as ‘1’.
  2. Setting the second MSB: The SAR register keeps the MSB as ‘1’ and sets the second most significant bit (bit 4) to ‘1’. The SAR value is now 11000 (binary), which is 24 (decimal). The DAC generates a reference voltage U0=24×1 mV=24 mV. The comparator compares U1=21.5 mV and U0=24 mV. Since U1<U0, the current bit is reset to ‘0’.
  3. Setting the third bit: The SAR register keeps the first two bits as ‘10’ and sets the third bit (bit 3) to ‘1’. The SAR value is now 10100 (binary), which is 20 (decimal). The DAC generates a reference voltage U0=20×1 mV=20 mV. The comparator compares U1=21.5 mV and U0=20 mV. Since U1>U0, the current bit is kept as ‘1’.
  4. Setting the fourth bit: The SAR register keeps the first three bits as ‘101’ and sets the fourth bit (bit 2) to ‘1’. The SAR value is now 10110 (binary), which is 22 (decimal). The DAC generates a reference voltage U0=22×1 mV=22 mV. The comparator compares U1=21.5 mV and U0=22 mV. Since U1<U0, the current bit is reset to ‘0’.
  5. Setting the LSB: The SAR register keeps the first four bits as ‘1010’ and sets the least significant bit (bit 1) to ‘1’. The SAR value is now 10101 (binary), which is 21 (decimal). The DAC generates a reference voltage U0=21×1 mV=21 mV. The comparator compares U1=21.5 mV and U0=21 mV. Since U1>U0, the current bit is kept as ‘1’.
  6. Result: After successive approximation, the SAR register value is 10101 (binary), which is 21 (decimal).

ADC Driver

Write an ADC driver, create an ADC device node via a misc device, and read the ADC value of the corresponding channel through this node.

API Function

iio_channel_get()

Function Prototype

Core Function

From the Linux IIO subsystem, based on the ‘IIO device name’ and ‘channel name’, obtain a correspondingstruct iio_channelchannel structure pointer; all subsequent operations on this ADC channel depend on this pointer.

Parameter Description

  • name: Specifies the IIO device name / ID, ifNULLis passed, it indicates using the system default IIO device; commonly used in SARADC scenariosNULL
  • channel_name: Specifies the name of the ADC channel to open; the specific name must match the kernel driver.

Return value

  • ✅ Success: returns a validstruct iio_channel *struct pointer (channel handle);
  • ❌ Failure: returns a kernel error pointerERR_PTR(error code), and cannot be used directly as a normal pointer.

iio_channel_release()

The acquired channel handlemust be released after use, otherwise it will cause a kernel resource leak. The release function is fixedly paired with:

  • Parameter: which isiio_channel_getthe acquired channel pointer
  • Feature: no return value, just call it safely.

struct iio_channel

iio_read_channel_raw()

Function prototype

Core function

Read from the opened IIO channel (ADC channel)Raw integer data without any calibration or conversion, which is the core function for reading ADC data.

✔️ Your SARADC scenario: The readvalis the voltage corresponding to the ADC pin acquisitionraw digital value(For example, a 10-bit ADC will have a value of 0~1023).

Parameter description

  • channel: Input parameter,iio_channel_getthe channel handle returned upon success,must be a valid pointer；
  • val: Output parameter, an integer pointer. After the function executes successfully, the collected ADC raw data will be written to the variable pointed to by this address.

Return value

  • ✅ Success: Returns0；
  • ❌ Failure: ReturnsNegative kernel error code(e.g., -EINVAL invalid parameter, -ENODEV device not found, etc.).

Key points to note

The readvalis raw bare data：

  • No unit, no calibration, no conversion, just a pure digital value after ADC analog-to-digital conversion;
  • The value range is determined by the ADC bit depth (e.g., 8-bit ADC: 0~255; 10-bit ADC: 0~1023; 12-bit ADC: 0~4095);
  • If you need to convert it to an actual voltage value, you need to write the formula yourself:Actual voltage = (raw value / ADC maximum range) * ADC reference voltage。
