This article introduces the fifth-generation hardware cache coherency protocol CHI in the AMBA 5 architecture, elaborating in detail on its mesh structure, layered architecture, and packet-based communication mechanism designed specifically for high-performance multi-core systems. The article further discusses the differences between CHI and the ACE protocol, the functional classification of various node types (RN, HN, SN), the Flit transmission format and ID allocation mechanism at the link layer, as well as key technical details such as system address mapping and completion acknowledgments.

Reference documents:

Arm Architecture Reference Manual Armv8

https://developer.arm.com/documentation/ddi0487/fc/

CHI Bus

is the fifth-generation protocol defined in AMBA 5, an evolution of ACE, designed specifically for high-performance, multi-core processor systems. It supports more complex cache coherency management and large-scale system integration, making it suitable for large-scale multi-core system scenarios that require high performance and complex cache coherency management.

Document:

  1. AMBA 5 CHI Architecture Specification，issue E.b
  2. Arm CoreLink CI-700 Coherent Interconnect Technical Reference Manual
  • CHI (Coherent Hub Interface) is the next-generation hardware cache coherency protocol, designed to accommodate varying numbers of processors and peripherals
    • Small systems: Embedded
    • Medium systems: Mobile phones
    • Large systems: Data center
  • Cache coherency protocol is similar to ACE
  • Supports hierarchical design
    • Protocol layer
    • Transport layer
    • Link layer

Common bus connection structures

Mesh structure: CI-700 or CMN-600

CI-700 Controller Introduction

  • Supports up to 8 CPU clusters

  • Supports up to 12 crosspoints (XP): hardware components for routing or switching

    • CI-700 forms a mesh through XPs
    • Each XP can have up to 4 neighbor XPs (up, down, left, right)
    • Each mesh can support 4 device ports
    • Each device port can be used to connect cache-coherent masters (RN-F) or slave devices (SN-F)

  • Supports up to 8 RN-F interfaces for connecting CPU clusters, GPUs, accelerator cards, or other cache-equipped master devices

  • Supports up to 8 HN-F and a maximum of 32MB system cache

  • Supports up to 8 SN interfaces

Differences between ACE and CHI

  • Similarities:
    • Goal: hardware cache coherency solution
    • Adopts similar cache state transitions
    • Understanding of snoop transfer transactions is very similar
  • Differences
    • ACE uses a crossbar structure, CHI uses a mesh structure
    • CHI adopts a layered design: protocol layer, transport layer, link layer
    • CHI uses packet-based communication
    • CHI uses the concepts of request node, home node, and slave node to describe transfer transactions
    • CHI supports more snoop transfer transactions
    • CHI supports DCT, DMT, DWT, etc. to optimize transfer performance
    • CHI supports atomic operations and cache stash

Node

RN (request node): request node

  • RN-F: cache-coherent request node
    • Built-in hardware cache-coherent cache
    • Allows generation of all transfer transactions
    • Supports all snoop transfers
  • RN-D: IO cache-coherent request node with built-in DVM (similar to ACE-Lite + DVM)
    • Does not include hardware cache coherence cache
    • Receive DVM transactions
    • Can generate a portion of transactions
  • RN - I: IO Request Node
    • Does not include hardware cache coherence cache
    • Does not receive DVM transactions
    • Can generate a portion of transactions
    • Does not support snoop

HN (Home Node): Master node on the system bus, used to receive transactions from request nodes

  • HN - F: Cache coherent master node
    • Receive all request types
    • Manage snoop requests from RN - F using PoC
    • Manage the ordering of memory requests using PoS
    • Includes directory or snoop filter to reduce redundant snoops
  • HN - I: Master node that does not support cache coherence
    • Can only process a portion of requests
    • Does not include POC, cannot process snoopable requests,
    • POS to handle the ordering of IO requests

SN: Slave node

  • **SN - F：**Slave device using normal memory, which can handle non-snoopable read/write, atomic requests (exclusive requests), and CMO requests
  • SN - I: Similar to SN - F, used for peripherals or normal memory

Channel

  • The channel defined by CHI is completely different from ACE.

  • The handshake protocol in Channel is different from AXI/ACE.
    • The FLITV signal is pulled high, indicating that the transmitter is ready to send a data packet, and the packet is valid.
    • The LCDRV signal being pulled high indicates that the receiver sends a credit to the transmitter: you can send.

Link layer

  • The link layer provides a streamlined mechanism for packet-based communication between nodes and interconnect IPs.
  • Provides a two-way link transmission mode.
    • Transmitter -> Receiver
    • Receiver -> Transmitter

Packet format - Flits

  • Flit = Flow control unIT, which is the smallest unit of transmission at the link layer. A packet contains multiple Flits.
    • Protocol Flit: Used to transmit protocol information
    • Link Flit: Used to transmit link maintenance information
  • CHI uses the form of protocol messages to transmit information, including various IDs, opcodes, memory attributes, addresses, data, error responses, etc.

Protocol flit

  • CHI defines 4 types of protocol flits.
    • Request flit
    • Response flit
    • Snoop flit
    • Data flit
  • Each type of flit has its own format.

ID

Many IDs are defined in the CHI protocol:

  • Source ID (SrcID): Indicates the ID of the sending node of the flit packet.
  • Target ID (TgtID): Indicates the ID of the target node receiving the flit packet.
  • Transaction ID (TxnID): Each transmission transaction has a unique ID, which can be used for outstanding requests, supporting a maximum of 256 outstanding transaction IDs. Similar to the transaction ID in AXI.
  • Request opcode (Opcode): Used to specify the transmission type.
  • Data Buffer ID (DBID): Used for responses and data packets, allowing the Completer of a transaction to provide its own identifier for the transaction.

ID assignment and binding

  • CHI uses the System Address Map (SAM) to convert the physical address in a transmission transaction into a target Node ID.
  • Each RN and HN has a SAM.
  • The CHI specification does not define how SAM is implemented, including the format and structure of SAM.
  • CHI requirements for SAM:
    • Describes the system-wide address space; all SAMs must be globally consistent, for example, address 0xFF00_0000 must map to the same HN.
    • For unmapped addresses, an error response mechanism must be provided.

Completion acknowledgement

  • Similar to the RACK and WACK signals in ACE, used to guarantee the ordering of transactions.
  • CompACK guarantee: HN-F can only process other snooping transactions after receiving the CompACK.
  • For read transactions:
    • Except for ReadNoSnp and ReadOnce* transactions, all other read transactions require CompACK.
    • RN-F will only send CompACK after receiving signals such as Comp, CompData, RespSepData, etc.
    • HN-F must wait for CompACK before sending snoops for other request transactions to the same address.
  • For write transactions:
    • Only WriteUnique and WriteNoSnp transactions require CompACK.

exclusive access

  • The Exclusive flow is similar to that in ACE.
  • The flow of Exclusive access:
    • Execute exclusive load
    • Calculate
    • Execute exclusive store
      • If another master writes to this address, it fails.
      • If no other master writes to this address, success
  • An LP (Logical Processor) monitor must be implemented on the RN-F (master) side.
  • A PoC (Point of Coherence) monitor must be implemented on the HN-F node within the CHI interconnect.

The LP monitor is located at RN-F:

  1. Each RN-F must implement an exclusive monitor to observe and monitor the memory address being exclusively accessed.
  2. When the CPU initiates an exclusive load, the LP monitor is set.
  3. The LP monitor is reset in the following cases:
    • If the address is overwritten by another LP
    • If the LP performs another store operation to this address

The PoC monitor is located at HN-F:

  1. The POC monitor records the snoop transactions for every exclusive access executed by an LP.
  2. The monitor monitors the exclusive accesses of all LPs in parallel.
  3. When the HN-F receives an exclusive load or store operation, the monitor registers this information: a certain LP is attempting an exclusive access.
  4. After an LP fails to execute an exclusive store, the LP needs to restart the access sequence of exclusive loads and stores.
  5. When the HN-F receives an exclusive store operation:
    • If the exclusive access record corresponding to this address has already been registered in the PoC monitor, and it has not been reset by another LP, then the exclusive store will succeed, and then all other access records attempting exclusive access will be reset.
    • If an LP executes an exclusive access, but it is not found in the POC monitor, then the exclusive store will fail.

Atomic access

  • Atomic access is added in the CHI.B protocol.
  • Atomic access allows operations and computations to be performed close to the data.
    • HN-F or SN contains ALU logic computation units.
  • Benefits of Atomic access:
    • More accurate and predictable latency.
    • No need to compete for the cache with other requesters, reducing memory access blocking and cache thrashing.
    • Fairness. When multiple requesters access a memory address simultaneously, arbitration is performed through POS or POC.
  • There are four types of Atomic transactions.
    • AtomicStore
    • AtomicLoad
    • AtomicSwap
    • AtomicCompare

New features.

cache stash

  • IO devices directly write data into the cache of the target RN-F.
  • Similar to Intel DDIO technology.
  • There are 4 types of transactions supported by Cache stash.
    • WriteUniquePtlStash
    • WriteUniqueFullStash
    • WriteUniqueFullStash
    • StashOnceShared

DMT and DCT.

  • In the CHI.A protocol, read data and snoop data must first be sent to the master node before being sent to the receiver node.
    • Disadvantage: increased transmission length and latency.
