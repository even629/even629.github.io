QEMU Address Space Abstraction

This article introduces the abstraction mechanism of address spaces in QEMU, discussing in detail the CPU's memory access flow and the two core concepts—address-space and memory-region—that QEMU provides to simulate memory and peripheral behavior. It further summarizes the address space layout, memory-region address overlap and alias mechanisms, the structure definitions and initialization processes of AddressSpace and MemoryRegion, and clarifies how QEMU uses these mechanisms to achieve discrete mapping and remapping of addresses, thereby constructing the physical address space view of a virtual machine.

Basic Introduction

From the CPU’s perspective, all memory access operations are performed on addresses (load/store). The CPU does not care what device corresponds to the address behind it, as long as it can read and write the correct results.

CPU Memory Access Flow

  1. After the CPU calculates the target address (via the Arithmetic Logic Unit, ALU), it sends it to the address bus, and simultaneously the CPU provides read/write control signals;
  2. **The device corresponding to the address may be a piece of regular memory or an I/O device (**here specifically referring to peripherals), which will respond to the signals on the address bus;
  3. If it is a read operation, the data corresponding to the address is transmitted back via the bus according to the bit width specified by the CPU, typically stored in the register specified by the CPU’s memory access instruction;
  4. If it is a write operation, the data transmitted from the bus is written to the specified address according to the bit width specified by the CPU. If it is an I/O device, this typically updates the register corresponding to the address and may produce side effects.

QEMU simulates memory and peripherals

To simulate the behavior of memory/peripherals, QEMU must at least implement the following mechanisms:

  1. Basic address space management,Ability to distinguish which device it is based on the address sent by the CPU；
  2. Implement discrete address mapping, as some peripheral addresses may not be contiguous；
  3. Implement address remapping, for example, the RAM and XRAM of MCS-51 both start from address 0;

For this purpose, QEMU provides two concepts,address-spaceandmemory-region(hereinafter referred to as mr), the former is used to describe the mapping relationship of the entire address space (different components may see different address spaces), and the latter is used to describe the mapping rules within a certain address range in the address space.

Memory region address overlap

**mr supports address range overlap at the same level.**The overlapping parts are presented according to priority, with the higher-priority overlapping part serving as the access target. In (prio 0, type), the prio is followed by the priority. Since there is no address overlap between virt peripherals, all priorities are 0.

alias example

  • alias memory-region maps a region to another address range in the address-space

  • alias is also a memory-region, but it internally references another region

AddressSpace

  • representsthe complete address space seen by the CPU or bus
  • includes:
    • all mappedmemory-region
    • priority (prio) of each region
    • type (RAM / ROM / I/O / alias)
  • can be understood asthe “physical address space view” of the virtual machine

a CPU can have multiple address-spaces (for example, a RISC-V CPU hascpu-memory-0, and other I/O spaces or PCI bus spaces).

MemoryRegion

  • of memory or I/OSpecific block
  • Includes:
    • Starting address range (offset relative to address-space)
    • Size
    • Type (RAM / ROM / I/O / alias / container)
    • Sub memory-region (supports nesting)
    • Corresponding read/write handler or object pointer

Can be understood asa “single block” in the address-space, which can be physical memory, device registers, PCI BAR, etc.
