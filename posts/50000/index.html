This article introduces the syntax and pseudo-instruction usage of the GNU AS assembler under the ARM64 (AArch64) architecture, covering labels, comments, symbols (global and local), alignment pseudo-instructions, data definition pseudo-instructions (integers, floating-point numbers, strings), repeat block definitions (.rept/.endr), constant definitions (.equ/.set), function-related pseudo-instructions, conditional assembly pseudo-instructions, and section-related pseudo-instructions (.section, .pushsection/.popsection), and compares the differences between AT&T syntax and ARM official assembly syntax.

Reference documents:

gas

https://sourceware.org/binutils/docs/as.pdf

ARM64 assembler

  • ARM’s official assembler
  • GNU AS assembler: aarch64-linux-gnu-as
  • gcc uses as as its assembler, so the assembly code is AT&T
    • AT&T: originating from Bell Labs, an assembly syntax created for developing the UNIX system
    • ARM format: ARM official assembly syntax

Syntax

  • label: Any identifier ending with a colon is considered a label
  • Comments：
    • // indicates a comment
    • # at the beginning of a line indicates a comment for the entire line
  • Instructions, pseudo-instructions, and registers can all be in uppercase or lowercase; GNU style defaults to lowercase.

Symbol

Represents its own address and can also be used as a variable or function.

  • Global symbol, can be declared with .global.
  • Local symbol, mainly used within a local scope, with label names starting directly with numbers 0-99, often used in conjunction with the b instruction.
  • f: Instructs the compiler to search forward.
  • b: Instructs the compiler to search backward.

Alignment pseudo-instruction.

  • .align aligns data by padding to achieve alignment. Can pad with 0 or use nop instructions.
    • Tells the assembler that the code following .align must be allocated starting from the next address divisible by 2^n.
    • In the ARM64 system, the first parameter represents the size as 2^n.

Pseudo-operations related to sections
  • .section indicates which section the following assembly will be linked into, such as code section, data section, etc.
  • Each section starts with a section name and ends with the next section name or the end of the file

Flags can be added afterward to indicate the attributes of the section

FlagMeaning Description
aallocatable: This section needs to be loaded into memory at runtime.
dGNU_MBIND section: A special binding section used by GNU.
eexcluded: This section will not be included in the executable file or shared library.
wwritable: This section is writable.
xexecutable: This section contains executable code.
Mmergeable: Can be merged with other sections having the same attributes (commonly used for read-only strings, etc.).
Sstring: This section contains null-terminated strings.
Ggroup: This section belongs to a section group (e.g., COMDAT).
Tthread-local-storage: This section is used for thread-local storage (TLS).
?unspecified group: This section belongs to the group of the previous section (if any).

Example:

.pushsection <name>
Place the following code or datainto the specified section, whilesaving the current section state。

.popsection
indicatesending the previous push, andRestore the original section。

  • Used in pairs。

  • Only affects the code betweenpushsectionandpopsection, and has no effect on other code.

  • The remaining code still belongs to the original section, for example.textor.data。

macro
  • .macro and .endm form a macro
  • .macro is followed by the macro name, and then the macro parameters
  • To use parameters in a macro, you need to add the prefix ""

Defined a macro named plus1 with two parameters p and p1

Using parameters in a macro requires a prefix; “\p” represents the first parameter, “\p1” is the second parameter

  • Macro parameters can be set with an initial value when defined

The first parameter p1 has an initial value of 0. In this case, you can use reserve_str a,b or reserve_str, b to call this macro

Solution:

  • Use spaces or use altmacro+&

  • Use “\()” to indicate concatenation

ARM64-specific features

ARM64 compilation options
  • -EB: for big-endian CPU, -EL: for little-endian CPU
  • -mabi: specify ABI mode, ilp32 for ELF32, lp64 for ELF64, default is lp64
  • -mcpu=processor+extension: specify CPU model, e.g., cortex-a72
  • -march=: specify supported architecture, e.g., armv8.2-a
  • Extensions supported by ARM64, see GNU assembler as_v2.34 Chapter 9.1.2
Special characters
  • // indicates a comment
  • # at the beginning of a line indicates a comment; otherwise, it can indicate an immediate value
  • :low12 indicates the lower 12 bits
  • ldr pseudo-operation
  • .bss switch to the bss section
  • .dword/.xword 64-bit data
  • name .reg register_name Name the register
